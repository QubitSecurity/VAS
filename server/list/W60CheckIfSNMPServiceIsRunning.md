KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드에 있는 **W-60 (W60CheckIfSNMPServiceIsRunning)** 항목은 **"SNMP 서비스 실행 여부 점검"**을 의미합니다.

한마디로 요약하면 **"서버의 상태를 원격에서 감시하는 '관리 기능(SNMP)'을 사용하지 않는다면, 해커에게 서버 내부 정보를 줄줄이 읊어주지 않도록 꺼버려라"**는 뜻입니다.

상세한 의미와 보안 위험성, 조치 방법은 다음과 같습니다.

---

### 1. 핵심 의미: "서버 상태 브리핑 기능"

**SNMP (Simple Network Management Protocol)**는 네트워크 장비(라우터, 스위치)나 서버의 상태(CPU 사용량, 트래픽, 디스크 용량 등)를 중앙 관제 시스템(NMS)에서 모니터링하기 위해 사용하는 프로토콜입니다.

* **SNMP 켜짐 (정보 제공):** 누군가(관리자 또는 해커)가 "너 지금 상태가 어때?"라고 물어보면, 서버는 "내 CPU는 50%고, 하드디스크는 80% 찼고, 실행 중인 프로세스는 A, B, C야"라고 대답해 줍니다.
* **SNMP 꺼짐 (정보 차단):** 외부에서 상태를 물어봐도 응답하지 않습니다.

### 2. 점검 목적 (Why?)

"내 서버 상태 좀 알려주는 게 왜 위험한가요?"

문제는 이 SNMP가 **"너무 솔직하고 말이 많다"**는 점입니다. 특히 설정이 미흡할 경우 치명적인 정보가 노출됩니다.

1. **정보 노출 (Reconnaissance):**
* 해커는 `snmpwalk` 같은 도구를 이용해 서버의 **운영체제 버전, 설치된 소프트웨어 목록, 실행 중인 프로세스, 열려 있는 포트, 심지어 사용자 계정 목록**까지 알아낼 수 있습니다.


2. **기본 패스워드(Community String) 문제:**
* SNMP는 `Community String`이라는 비밀번호를 사용하는데, 전 세계 공통 기본값이 **"public" (읽기 전용)**과 **"private" (쓰기 가능)**입니다.
* 관리자가 이를 바꾸지 않고 서비스를 켜두면, 해커는 "public"이라고만 입력하면 서버의 모든 정보를 가져갈 수 있습니다.


3. **서비스 거부 및 설정 변조:**
* 만약 "private" (쓰기 권한) 스트링까지 노출되면, 해커가 원격에서 서버 설정을 바꾸거나 장비를 재부팅시켜 버릴 수 있습니다.



!

### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 판단 기준은 다음과 같습니다.

* **양호 (Safe):**
* SNMP 서비스를 **사용하지 않아 중지(Disable)** 시킨 경우
* 만약 사용해야 한다면, Community String을 복잡하게 설정하고 **접근 가능한 IP(ACL)를 제한**한 경우


* **취약 (Vulnerable):**
* SNMP 서비스를 사용하지 않는데도 **실행 중(Running)**인 경우
* 사용 중이지만 Community String이 **"public", "private"** 등 기본값으로 설정된 경우



### 4. 설정 확인 및 조치 방법

서버 관리 목적상 반드시 필요한 경우가 아니라면, 서비스를 끄는 것이 가장 안전합니다.

#### 방법 A. 서비스 중지 (권장)

1. **실행 창 열기:** `Win + R` > `services.msc` 입력
2. **서비스 찾기:** **"SNMP Service"** 항목을 찾습니다.
3. **조치:**
* 더블 클릭 후 시작 유형을 **[사용 안 함(Disabled)]**으로 변경하고, 서비스 상태를 **[중지(Stop)]**로 설정합니다.



#### 방법 B. 보안 설정 강화 (사용해야 할 경우)

만약 모니터링 시스템(NMS) 때문에 꼭 써야 한다면, 다음 설정을 적용해야 합니다.

1. **서비스 속성:** `services.msc` > `SNMP Service` 더블 클릭
2. **보안(Security) 탭 이동:**
3. **커뮤니티 이름 변경:**
* 기본값인 `public`을 제거하고, 유추하기 어려운 복잡한 문자열(암호)로 새로 추가합니다.


4. **호스트 제한 (IP 필터링):**
* "모든 호스트로부터의 SNMP 패킷 받아들이기"를 체크 해제하고,
* **"다음 호스트로부터의 SNMP 패킷 받아들이기"**를 선택한 뒤, **실제 관제 서버의 IP 주소**만 등록합니다.



### 요약

> **W-60 항목은 "안 쓰는 '수다쟁이(SNMP)' 서비스는 꺼라"는 뜻입니다.**
> 만약 꼭 써야 한다면, 아무나 말을 걸지 못하게 '비밀번호(Community String)'를 바꾸고 '아는 사람(IP)'하고만 대화하게 설정하세요.

이로써 IIS 웹 서버뿐만 아니라 윈도우 서버 시스템 전반의 핵심 보안 설정들을 살펴보았습니다. 혹시 **비밀번호 정책(W-01~W-04)**이나 **로그 관리(W-05)** 등 기초적인 부분도 다시 점검하고 싶으신가요?
