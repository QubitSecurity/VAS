KISA(한국인터넷진흥원)의 주요정보통신기반시설 취약점 분석·평가 가이드에 있는 **W-75 (W75ConfigureWarningMessage)** 항목은 **"로그온 시 경고 메시지 설정"**을 의미합니다.

한마디로 요약하면 **"로그인하기 직전에 '관계자 외 출입 금지'라는 경고창을 띄워라"**는 뜻입니다.

이 항목의 상세한 의미와 왜 이것이 보안 취약점 점검 항목에 포함되는지 설명해 드리겠습니다.

---

### 1. 핵심 의미: 법적 효력을 위한 "경고판"

윈도우 서버에 접속하려고 할 때, 아이디/비밀번호 입력창이 뜨기 전에 **확인(OK) 버튼을 눌러야만 넘어가는 팝업창**을 보신 적이 있나요? 바로 그 기능을 말합니다.

* **설정 안 함 (취약):** 아무런 경고 없이 바로 로그인 화면이 뜹니다.
* **설정 함 (양호):** "이 시스템은 인가된 사용자만 이용할 수 있으며, 불법 접속 시 처벌받을 수 있습니다."라는 내용의 창이 뜹니다.

### 2. 점검 목적 (Why?)

이 설정은 기술적으로 해킹을 막아주는 기능은 아닙니다. 하지만 **법적(Legal)**, **관리적** 측면에서 매우 중요합니다.

1. **법적 대응 근거 마련:**
* 만약 해커를 잡아서 재판을 하게 되었을 때, 해커가 **"나는 이게 들어오면 안 되는 서버인지 몰랐다"**거나 **"경고 문구가 없었으므로 내 접속은 불법이 아니다"**라고 발뺌하는 것을 방지합니다.
* 명시적인 경고문("Unauthorized access is prohibited")을 보여주고 사용자가 '확인'을 눌렀다는 것은, **자신의 행위가 불법임을 인지하고 동의했다**는 강력한 증거가 됩니다.


2. **심리적 억제:**
* 단순 호기심으로 접속하려던 내부 사용자나 비인가자에게 "모니터링 중이며 처벌받을 수 있다"는 메시지를 주어 접속 시도를 포기하게 만듭니다.



### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 판단 기준은 다음과 같습니다.

* **양호 (Safe):**
* 서버 로그온 시 적절한 **제목(Title)**과 **내용(Text)**이 담긴 경고창이 뜨는 경우


* **취약 (Vulnerable):**
* 경고 메시지 창이 뜨지 않거나, 내용이 비어있는 경우



### 4. 설정 확인 및 조치 방법

이 설정은 **로컬 보안 정책**의 보안 옵션에서 두 가지 항목을 모두 설정해야 합니다.

1. **실행 창 열기:** `Win + R` 키를 누르고 `secpol.msc` 입력
2. **경로 이동:**
* `로컬 정책(Local Policies)` > `보안 옵션(Security Options)`


3. **정책 설정 (두 가지 모두 설정 필수):**
* **① 대화형 로그온: 로그온을 시도하는 사용자에게 표시할 메시지 제목 (Message title)**
* 값 예시: `시스템 접근 경고`, `경고`, `Authorized Use Only`


* **② 대화형 로그온: 로그온을 시도하는 사용자에게 표시할 메시지 텍스트 (Message text)**
* 값 예시: (아래 예시 문구 참조)
