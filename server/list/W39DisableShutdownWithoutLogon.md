KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드에 있는 **W-39 (W39DisableShutdownWithoutLogon)** 항목은 **"로그인 전 시스템 종료 버튼 비활성화"**를 의미합니다.

한마디로 요약하면 **"아이디와 비밀번호를 입력하여 로그인하기 전에는, 컴퓨터를 끄거나 재부팅할 수 없도록 전원 버튼을 없애라"**는 뜻입니다.

상세한 의미와 보안 목적은 다음과 같습니다.

---

### 1. 핵심 의미: 로그인 화면의 전원 버튼

여러분이 윈도우 PC를 켤 때, 아이디와 암호를 입력하는 **로그인 대기 화면(Welcome Screen)** 우측 하단에 보통 **전원 아이콘(시스템 종료/다시 시작)**이 있는 것을 보셨을 겁니다.

* **취약한 상태 (허용됨):** 로그인하지 않은 상태에서도 누구나 이 버튼을 눌러 서버를 꺼버릴 수 있습니다.
* **안전한 상태 (허용 안 함):** 이 버튼이 아예 보이지 않거나 비활성화되어, **반드시 로그인을 성공한 사용자만** 시스템을 종료할 수 있습니다.

### 2. 점검 목적 (Why?)

이 설정이 중요한 이유는 두 가지 보안 원칙 때문입니다.

1. **가용성 (Availability) 보장:**
* 서버는 항상 켜져 있어야 합니다. 그런데 지나가던 사람이나 실수로 키보드/마우스를 건드린 사람이 로그인 화면에서 서버를 꺼버린다면 서비스 장애가 발생합니다.
* 악의적인 목적을 가진 사람이 물리적으로 접근하여(또는 원격 데스크톱 연결 화면에서) 서버를 계속 재부팅 시켜 서비스를 방해하는 것을 막습니다.


2. **책임 추적성 (Accountability) 확보:**
* 로그인을 한 후에 종료하면, 시스템 로그(Event Log)에 **"누가(예: Administrator)"** 시스템을 껐는지 기록이 남습니다.
* 하지만 로그인 없이 끄면, "누가" 껐는지 알 수 없고 단순히 시스템이 종료되었다는 기록만 남게 되어, 사고 원인을 파악하기 어렵습니다.



### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인 및 윈도우 보안 정책에 따른 판단 기준은 다음과 같습니다.

* **양호 (Safe):** 정책 값이 **"사용 안 함(Disabled)"** 또는 **"0"**으로 설정된 경우
* *(의미: 로그인하지 않고 끄는 것을 허용하지 않음)*


* **취약 (Vulnerable):** 정책 값이 **"사용(Enabled)"** 또는 **"1"**로 설정된 경우
* *(의미: 로그인하지 않고도 끌 수 있게 허용함)*



### 4. 설정 확인 및 조치 방법

이 설정은 **로컬 보안 정책**에서 제어합니다.

1. **실행 창 열기:** `Win + R` 키를 누르고 `secpol.msc` 입력
2. **경로 이동:**
* `로컬 정책(Local Policies)` > `보안 옵션(Security Options)`


3. **정책 찾기:**
* **"시스템 종료: 로그온하지 않고 시스템 종료 허용 (Shutdown: Allow system to be shut down without having to log on)"** 항목을 찾습니다.


4. **조치:**
* 해당 항목을 더블 클릭하여 **[사용 안 함(Disabled)]**으로 설정하고 확인합니다.



### 요약

> **W-39 항목은 "서버를 끄려면 반드시 로그인부터 하라"는 규칙입니다.**
> 이렇게 해야 아무나 서버를 끄는 사고를 막고, 만약 꺼졌더라도 누가 껐는지 범인을 찾을 수 있습니다.

현재 운영 중인 서버가 IDC(데이터 센터)에 있더라도, KVM 스위치나 콘솔 등을 통해 접근 가능한 상황을 고려하여 이 설정은 **"사용 안 함"**으로 두는 것이 원칙입니다.
