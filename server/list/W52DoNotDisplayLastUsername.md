KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드에 있는 **W-52 (W52DoNotDisplayLastUsername)** 항목은 **"로그인 화면에 마지막 사용자 이름 표시 안 함"**을 의미합니다.

한마디로 요약하면 **"로그인 화면에 직전에 사용했던 사람의 아이디(ID)가 자동으로 뜨지 않게 하여, 해커에게 아이디 정보를 노출하지 말라"**는 뜻입니다.

상세한 내용과 보안 목적은 다음과 같습니다.

---

### 1. 핵심 의미: 친절함 vs. 보안

윈도우는 기본적으로 사용자의 편의를 위해, 직전에 로그인했던 사용자의 이름을 기억했다가 다음 로그인 화면에 미리 보여줍니다.

* **취약한 상태 (이름 표시됨):** 로그인 화면에 `Administrator` 또는 `User1`이라는 아이디가 이미 입력되어 있거나 아이콘으로 떠 있습니다. 사용자는 비밀번호만 입력하면 됩니다.
* **안전한 상태 (이름 숨김):** 로그인 화면이 **텅 비어 있습니다.** 사용자는 **아이디와 비밀번호를 둘 다 직접 입력**해야 합니다.

### 2. 점검 목적 (Why?)

"아이디 좀 보여주는 게 왜 보안 위협인가요?"라고 생각할 수 있지만, 이는 해커에게 **절반의 성공**을 선물하는 것과 같습니다.

1. **공격 난이도 상승:** 로그인 인증은 `아이디` + `비밀번호` 조합을 맞춰야 성공합니다.
* 아이디가 노출되면: 해커는 **비밀번호**만 알아내면 됩니다. (공격 대상이 명확해짐)
* 아이디가 숨겨지면: 해커는 **아이디와 비밀번호를 모두** 추측해야 합니다. (공격 난이도가 훨씬 높아짐)


2. **사회 공학적 공격 방지:** 특정 직원의 아이디(이름)를 알게 되면, 이를 이용해 해당 직원을 사칭하거나 피싱 공격을 시도할 수 있습니다.

### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 판단 기준은 다음과 같습니다.

* **양호 (Safe):** 해당 정책이 **"사용(Enabled)"**으로 설정된 경우
* *(의미: 마지막 사용자 이름을 표시하지 않음 기능을 '켠다')*


* **취약 (Vulnerable):** 해당 정책이 **"사용 안 함(Disabled)"**으로 설정된 경우
* *(의미: 마지막 사용자 이름을 표시하지 않음 기능을 '끈다' → 즉, 이름을 표시한다)*



### 4. 설정 확인 및 조치 방법

이 설정은 **로컬 보안 정책**의 보안 옵션에서 제어합니다.

1. **실행 창 열기:** `Win + R` 키를 누르고 `secpol.msc` 입력
2. **경로 이동:**
* `로컬 정책(Local Policies)` > `보안 옵션(Security Options)`


3. **정책 찾기:**
* **"대화형 로그온: 마지막 사용자 이름 표시 안 함 (Interactive logon: Do not display last signed-in)"** 항목을 찾습니다.
* *(구버전 윈도우에서는 "마지막 사용자 이름 표시 안 함"으로 나올 수 있습니다.)*


4. **조치:**
* 해당 항목을 더블 클릭하여 **[사용(Enabled)]**으로 설정하고 확인합니다.



### 요약

> **W-52 항목은 "로그인 화면을 백지 상태로 만들어라"는 뜻입니다.**
> 해커나 제3자가 모니터만 보고도 "아, 이 서버의 관리자 아이디는 'admin'이구나"라고 알 수 없도록 막는 기본적인 보안 조치입니다.

이 설정을 적용하면 매번 아이디를 입력해야 해서 조금 귀찮을 수 있지만, 서버 보안을 위해서는 필수적인 설정입니다. 혹시 **"W-53 로그온 시도 실패 시 로그온 정보 표시 제한"** 같은 유사한 항목에 대해서도 궁금하신 점이 있으신가요?
