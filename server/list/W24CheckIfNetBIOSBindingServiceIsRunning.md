KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드에 있는 **W-24 (W24CheckIfNetBIOSBindingServiceIsRunning)** 항목은 **"NetBIOS 바인딩 서비스 실행 여부 점검"**을 의미합니다.

한마디로 요약하면 **"서버가 네트워크상에 '내 이름은 OOO이고, 나는 여기 있어요!'라고 동네방네 소리치고 다니는 기능(NetBIOS)을 꺼라"**는 뜻입니다.

상세한 의미와 보안 위험성, 조치 방법은 다음과 같습니다.

---

### 1. 핵심 의미: "수다쟁이 구형 프로토콜"

**NetBIOS (Network Basic Input/Output System)**는 아주 오래전(DOS, Windows 95 시절)에 컴퓨터끼리 통신하기 위해 만들어진 프로토콜입니다.
이 기능이 켜져 있으면, 컴퓨터는 자신의 이름(Computer Name)과 소속 그룹(Workgroup) 정보를 네트워크 전체에 **브로드캐스트(방송)**합니다.

* **TCP/IP 바인딩 됨 (취약):** 네트워크 카드(NIC)가 TCP/IP 통신을 할 때 NetBIOS 기능도 같이 사용합니다. (포트 137, 138, 139 사용)
* **TCP/IP 바인딩 해제 (양호):** 최신 통신 방식(DNS, SMB over TCP - 포트 445)만 사용하고, 구형 NetBIOS는 사용하지 않습니다.

### 2. 점검 목적 (Why?)

"그냥 이름 좀 알려주는 게 왜 위험한가요?"

해커가 내부 네트워크에 침투했을 때, NetBIOS는 **가장 먼저 악용되는 먹잇감**이기 때문입니다.

1. **정보 노출 (Information Leakage):**
* 해커는 아무런 권한 없이도 네트워크상의 모든 컴퓨터 이름, 사용자 그룹, OS 버전 등의 정보를 수집할 수 있습니다.


2. **스푸핑 공격 (Spoofing & Poisoning):**
* **LLMNR/NBT-NS Poisoning:** 해커가 NetBIOS 요청을 중간에서 가로채서, "내가 네가 찾는 그 서버야"라고 거짓말을 합니다.
* 이를 통해 사용자의 **로그인 해시(암호화된 비밀번호)**를 탈취할 수 있습니다. (Responder 같은 해킹 툴이 이 원리를 이용합니다.)


3. **불필요한 트래픽:** 끊임없이 이름 확인 패킷을 날리기 때문에 네트워크 대역폭을 낭비합니다.

### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 판단 기준은 다음과 같습니다.

* **양호 (Safe):** TCP/IP 속성에서 **"NetBIOS over TCP/IP 사용 안 함(Disable)"**으로 설정된 경우
* **취약 (Vulnerable):** "기본값" 또는 "사용(Enable)"으로 설정되어 있어, NetBIOS 포트(137~139)가 열려 있는 경우

### 4. 설정 확인 및 조치 방법 (네트워크 설정)

이 설정은 네트워크 어댑터(LAN 카드) 설정 깊숙한 곳에 숨어 있습니다.

1. **실행 창 열기:** `Win + R` > `ncpa.cpl` 입력 (네트워크 연결 창)
2. **어댑터 속성:** 사용 중인 이더넷(Ethernet) 아이콘 우클릭 > **[속성(Properties)]**
3. **IPv4 속성:** `인터넷 프로토콜 버전 4(TCP/IPv4)` 선택 > **[속성(Properties)]**
4. **고급 설정:** 우측 하단의 **[고급(Advanced)]** 버튼 클릭
5. **WINS 탭 이동:** 상단의 **[WINS]** 탭 클릭
6. **조치 (끄기):**
* 하단의 NetBIOS 설정에서 **[NetBIOS over TCP/IP 사용 안 함(Disable NetBIOS over TCP/IP)]** 라디오 버튼을 선택합니다.
* 확인을 눌러 저장합니다.



### 💡 주의사항: 파일 공유가 안 되면 어떡하죠?

"이걸 끄면 파일 공유(폴더 공유)가 안 되나요?"
 **대부분의 경우 문제없이 잘 됩니다.**

* **최신 윈도우:** Windows 2000 이후의 모든 윈도우는 파일 공유 시 **TCP 445번 포트(Direct SMB)**를 사용하므로, NetBIOS(139번)를 꺼도 파일 공유는 정상 작동합니다.
* **예외 상황:** 만약 회사 내에 **아주 오래된 장비(Windows 98, NT 4.0 등)**나 **구형 복합기**가 있어서 이름으로 접속해야 한다면 연결이 끊길 수 있습니다. 이 경우에만 예외적으로 켜둡니다.

### 요약

> **W-24 항목은 "이제는 쓸모없고 위험하기만 한 '옛날 통신 방식'을 꺼라"는 뜻입니다.**

이 설정을 적용하면 네트워크가 조용해지고, 해커가 내부망 정보를 수집하기 훨씬 어려워집니다.

이것으로 KISA 가이드의 윈도우 서버 주요 보안 설정(계정, 서비스, IIS, 네트워크 등)을 폭넓게 살펴보았습니다. 혹시 리눅스(Unix) 계열 서버 점검 항목이나, 방화벽/네트워크 장비 점검에 대해서도 도움이 필요하신가요?
