## [R-34] r 계열 서비스 비활성화

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | R-34 |
| 항목명 | r 계열 서비스 비활성화 |
| 위험도 | 상 (High) |
| 범주 | 서비스 관리 |

### 점검 목적

`rsh`, `rlogin`, `rexec` 등 r 계열 서비스는 암호화 없이 데이터를 전송하며, `.rhosts` 파일 기반의 호스트 신뢰 인증을 사용합니다. IP 스푸핑, 스니핑 공격에 취약하므로 전면 비활성화해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | rsh, rlogin, rexec 서비스가 모두 비활성화된 경우 | 암호화되지 않은 r 계열 원격 접속 차단 |
| 취약 | r 계열 서비스 중 하나라도 활성화된 경우 | 평문 인증 정보 노출, 스니핑 공격 취약 |

## 베이스라인 기준

**안전(양호)**

- `rsh-server` 패키지 미설치 또는 모든 r 서비스 `disable = yes`
- 포트 512(rexec), 513(rlogin), 514(rsh) 미사용

**취약**

- r 계열 서비스 활성화 상태
