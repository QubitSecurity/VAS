## [R-75] SSH 접근 가능 사용자/그룹 제한

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | R-75 |
| 항목명 | SSH 접근 가능 사용자/그룹 제한 |
| 위험도 | 상 (High) |
| 범주 | 서비스 관리 |

### 점검 목적

SSH 서비스에 접속 가능한 사용자/그룹을 명시적으로 제한하지 않으면 시스템의 모든 계정이 원격 접속을 시도할 수 있습니다. `AllowUsers` 또는 `AllowGroups` 지시어로 허용 대상을 최소화해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | AllowUsers 또는 AllowGroups로 접속 허용 대상이 명시된 경우 | 인가된 사용자/그룹만 SSH 접속 가능 |
| 취약 | AllowUsers/AllowGroups 설정이 없는 경우 | 모든 계정이 SSH 접속 시도 가능 |

## 베이스라인 기준

**안전(양호)**

- `AllowUsers admin deploy` 등 허용 사용자 명시
- 또는 `AllowGroups sshusers wheel` 등 허용 그룹 명시

**취약**

- `AllowUsers`, `AllowGroups` 설정 없음 (모든 계정 접속 허용)
