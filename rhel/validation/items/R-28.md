## [R-28] world writable 파일 점검

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | R-28 |
| 항목명 | world writable 파일 점검 |
| 위험도 | 상 (High) |
| 범주 | 파일 및 디렉토리 관리 |

### 점검 목적

모든 사용자가 쓰기 가능한(world writable) 파일은 악의적인 사용자가 내용을 변조하거나 악성 코드를 주입할 수 있습니다. 특히 실행 파일이나 설정 파일이 world writable인 경우 매우 심각한 보안 위협이 됩니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | world writable 파일이 없거나, Sticky bit가 설정된 공유 디렉토리만 존재하는 경우 | 임의 사용자의 파일 변조 불가 |
| 취약 | 불필요한 world writable 파일이 존재하는 경우 | 파일 변조, 악성 코드 주입 위험 |

## 베이스라인 기준

**안전(양호)**

- 불필요한 world writable 파일 없음
- `/tmp`, `/var/tmp` 등 공유 디렉토리에는 Sticky bit(`t`) 설정

**취약**

- 실행파일, 설정파일 등에 world writable 존재
