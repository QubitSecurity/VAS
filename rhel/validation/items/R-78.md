## [R-78] CGI 실행 제한 및 불필요한 모듈 비활성화

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | R-78 |
| 항목명 | CGI 실행 제한 및 불필요한 모듈 비활성화 |
| 위험도 | 상 (High) |
| 범주 | 서비스 관리 |

### 점검 목적

CGI 실행이 허용된 디렉토리에서는 임의의 스크립트가 서버 권한으로 실행될 수 있습니다. `ExecCGI` 옵션을 제거하고 `cgi_module`을 비활성화하여 CGI를 통한 원격 코드 실행을 차단해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | ExecCGI 옵션이 제거되고 cgi_module이 비활성화된 경우 | CGI 스크립트 실행 불가 |
| 취약 | ExecCGI 또는 cgi_module이 활성화된 경우 | 임의 스크립트 서버 권한으로 실행 가능 |

## 베이스라인 기준

**안전(양호)**

- Apache: `Options -ExecCGI`, `cgi_module` 비활성화(주석 처리)
- Nginx: 업로드 디렉토리 내 `.php`, `.cgi` 실행 차단 location 규칙 존재

**취약**

- `Options +ExecCGI` 또는 `Options All`
- `LoadModule cgi_module` 활성화 상태
