## [R-69] 보안 위협에 대한 위험 해소

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | R-69 |
| 항목명 | 보안 위협에 대한 위험 해소 |
| 위험도 | 중 (Medium) |
| 범주 | 보안 관리 |

### 점검 목적

SELinux 또는 기타 보안 기능이 비활성화되어 있으면 시스템의 접근 제어 기능이 크게 약화됩니다. Red Hat에서는 SELinux를 Enforcing 모드로 운영하여 추가적인 접근 제어 계층을 유지해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | SELinux가 Enforcing 모드로 활성화된 경우 | MAC 기반 추가 접근 제어 적용 |
| 취약 | SELinux가 Disabled 또는 Permissive 모드인 경우 | MAC 접근 제어 미적용, 취약점 익스플로잇 용이 |

## 베이스라인 기준

**안전(양호)**

- `getenforce` 결과: `Enforcing`
- `/etc/selinux/config`: `SELINUX=enforcing`

**취약**

- `SELINUX=disabled` 또는 `SELINUX=permissive`
