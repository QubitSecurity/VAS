## [U-63] 로그온 실패 로그 기록 및 확인

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | U-63 |
| 항목명 | 로그온 실패 로그 기록 및 확인 |
| 위험도 | 상 (High) |
| 범주 | 보안 관리 |

### 점검 목적

로그인 실패 기록을 남기지 않으면 무차별 대입 공격 탐지 및 침해 사고 분석이 불가능합니다. `/var/log/secure`, `/var/log/btmp`를 통해 로그인 실패 이력이 정상적으로 기록되고 있는지 확인해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 로그인 실패 로그가 정상 기록되고 정기 검토가 이루어지는 경우 | 무차별 대입 공격 탐지 가능 |
| 취약 | 로그인 실패 로그가 기록되지 않는 경우 | 공격 탐지 불가, 침해 사고 분석 어려움 |

## 베이스라인 기준

**안전(양호)**

- `/var/log/secure` 존재 및 `Failed` 로그 기록
- `/var/log/btmp` 존재 및 `lastb` 정상 동작
- rsyslog에 `authpriv.*` 로그 설정

**취약**

- 로그 파일 없음, 로그 기록 비활성화
