## [R-86] cron 작업 내 의심스러운 명령어 점검

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | R-86 |
| 항목명 | cron 작업 내 의심스러운 명령어 점검 |
| 위험도 | 중 (Medium) |
| 범주 | 보안 관리 |

### 점검 목적

cron에 등록된 작업은 지정된 주기로 자동 실행되므로 공격자가 악성 명령을 cron에 등록하여 지속성을 확보하는 방식으로 악용합니다. cron 파일의 내용을 주기적으로 검토하여 의심스러운 명령(역방향 쉘, 다운로더, 권한 상승 명령 등)을 찾아야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | cron 작업이 모두 업무 목적으로 확인되고 의심 패턴이 없는 경우 | 정상 자동화 작업만 등록됨 |
| 취약 | cron에 역방향 쉘, 다운로더, 출처 불명 스크립트 등이 등록된 경우 | 악성 명령 주기적 자동 실행 |

## 베이스라인 기준

**안전(양호)**

- 모든 cron 작업이 업무 목적으로 식별됨
- 역방향 쉘, 다운로더, 인코딩된 명령 없음

**취약**

- 역방향 쉘(`/dev/tcp`, `nc -e` 등) 명령 존재
- `wget`, `curl`을 통한 외부 스크립트 다운로드 후 실행
