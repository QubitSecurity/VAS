## [R-76] 원격 세션 타임아웃 (TMOUT) 설정

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | R-76 |
| 항목명 | 원격 세션 타임아웃 (TMOUT) 설정 |
| 위험도 | 중 (Medium) |
| 범주 | 서비스 관리 |

### 점검 목적

원격 접속 세션이 일정 시간 이상 유휴 상태로 유지될 경우, 자리를 비운 사이 타인이 해당 세션을 악용할 수 있습니다. 환경변수 `TMOUT`을 설정하여 일정 시간(권장 900초) 이상 입력이 없으면 자동으로 세션을 종료해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | TMOUT이 900 이하 양수로 설정된 경우 | 최대 15분 유휴 후 세션 자동 종료 |
| 취약 | TMOUT 미설정, 0, 또는 900 초과인 경우 | 유휴 세션 무기한 유지 |

## 베이스라인 기준

**안전(양호)**

- `/etc/profile` 또는 `/etc/profile.d/tmout.sh`에 `TMOUT=900` 이하 설정
- `readonly TMOUT` 및 `export TMOUT` 선언

**취약**

- TMOUT 미설정 (기본값 0 = 무제한)
- TMOUT=0 또는 900 초과
