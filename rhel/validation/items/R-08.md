## [R-08] 패스워드 최대 사용 기간 설정

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | R-08 |
| 항목명 | 패스워드 최대 사용 기간 설정 |
| 위험도 | 중 (Medium) |
| 범주 | 계정 관리 |

### 점검 목적

패스워드를 장기간 변경하지 않으면 이미 노출된 패스워드가 지속적으로 악용될 수 있습니다. 정기적인 패스워드 변경을 강제하여 크리덴셜 노출로 인한 피해 기간을 최소화해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 패스워드 최대 사용 기간이 90일 이하로 설정된 경우 | /etc/login.defs PASS_MAX_DAYS 설정 |
| 취약 | 패스워드 최대 사용 기간 설정이 없거나 90일을 초과하는 경우 | 패스워드 무기한 사용 허용 |

## 베이스라인 기준

**안전(양호)**

- `/etc/login.defs`: `PASS_MAX_DAYS 90` 이하
- 개별 계정 `chage -l` 결과 최대 사용 기간 90일 이하

**취약**

- `PASS_MAX_DAYS 99999` (무제한) 또는 90일 초과
