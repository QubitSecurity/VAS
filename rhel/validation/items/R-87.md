## [R-87] Samba 공유 및 익명 접근 제한

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | R-87 |
| 항목명 | Samba 공유 및 익명 접근 제한 |
| 위험도 | 상 (High) |
| 범주 | 보안 관리 |

### 점검 목적

Samba를 통한 파일 공유에서 익명(Guest) 접근이 허용되거나 공유 권한이 과도하게 설정되면, 인증 없이 파일 시스템에 접근하거나 민감 파일이 유출될 수 있습니다. 숨김 공유(`$` 접미사)와 게스트 접근을 차단하고 접근 제어를 강화해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 익명 접근이 비활성화되고 필요한 공유에만 명시적 접근 제어가 설정된 경우 | 인가된 사용자만 공유 접근 가능 |
| 취약 | guest ok = yes 또는 public = yes로 익명 접근이 허용된 경우 | 인증 없이 공유 파일 접근 가능 |

## 베이스라인 기준

**안전(양호)**

- `[global]` 섹션에 `restrict anonymous = 2` 설정
- `guest ok = no` (기본값 유지 또는 명시)
- 공유별 `valid users = <계정>` 명시
- 숨김 공유(`$`) 없음

**취약**

- `guest ok = yes` 또는 `public = yes`
- `[global]`에 `guest account = root`
