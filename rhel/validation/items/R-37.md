## [R-37] NFS 서비스 비활성화

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | R-37 |
| 항목명 | NFS 서비스 비활성화 |
| 위험도 | 상 (High) |
| 범주 | 서비스 관리 |

### 점검 목적

NFS(Network File System)는 네트워크를 통한 파일 시스템 공유 서비스입니다. 적절한 접근 제어 없이 활성화될 경우 내부 파일 시스템이 노출될 수 있으며, RPC 기반의 취약점이 악용될 수 있습니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | NFS 서비스가 비활성화되거나 운영에 필요한 경우 적절한 접근 제어가 설정된 경우 | 불필요한 파일 시스템 노출 없음 |
| 취약 | NFS 서비스가 활성화되고 접근 제어가 설정되지 않은 경우 | 내부 파일 시스템 무단 마운트 가능 |

## 베이스라인 기준

**안전(양호)**

- `nfs-server` 서비스 비활성화 또는 `/etc/exports`에 명시적 허용 IP만 설정

**취약**

- NFS 활성화 + `/etc/exports`에 `*(rw)` 등 무제한 허용
