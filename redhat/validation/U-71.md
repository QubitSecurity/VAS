## [U-71] FTP 서비스 보안 설정

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | U-71 |
| 항목명 | FTP 서비스 보안 설정 |
| 위험도 | 상 (High) |
| 범주 | 보안 관리 |

### 점검 목적

FTP는 인증 정보를 평문으로 전송하며, Anonymous FTP, 디렉토리 트래버설 등 다양한 보안 취약점이 있습니다. FTP 서비스를 반드시 운영해야 할 경우 SFTP(SSH 기반)나 FTPS(TLS 적용)로 대체하거나, vsftpd의 보안 설정을 강화해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | FTP 서비스가 비활성화되거나 보안 강화된 SFTP/FTPS를 사용하는 경우 | 평문 인증 정보 전송 차단 |
| 취약 | 보안 설정 없이 기본 FTP 서비스가 운영 중인 경우 | 평문 전송, Anonymous 허용 등 보안 위협 |

## 베이스라인 기준

**안전(양호)**

- FTP 미운영 시 비활성화
- 운영 시: `anonymous_enable=NO`, `chroot_local_user=YES`, SSL/TLS 적용

**취약**

- `anonymous_enable=YES`
- `chroot_local_user=NO` (상위 디렉토리 접근 가능)
- FTP 배너에 버전 정보 노출
