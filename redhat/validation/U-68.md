## [U-68] wtmp, utmp, btmp 로그 설정

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | U-68 |
| 항목명 | wtmp, utmp, btmp 로그 설정 |
| 위험도 | 상 (High) |
| 범주 | 보안 관리 |

### 점검 목적

`wtmp`(로그인 성공 이력), `utmp`(현재 로그인 사용자), `btmp`(로그인 실패 이력) 파일은 사용자 접속 추적에 필수적입니다. 이 파일이 없거나 권한이 잘못 설정되면 침해 사고 발생 시 사용자 행위 추적이 불가능합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | wtmp, utmp, btmp 파일이 존재하고 적절한 권한이 설정된 경우 | 사용자 접속 이력 완전 기록 |
| 취약 | wtmp, utmp, btmp 파일이 없거나 권한이 부적절한 경우 | 사용자 접속 추적 불가 |

## 베이스라인 기준

**안전(양호)**

- `/var/log/wtmp` 존재, 권한 `664` 또는 `644`
- `/var/run/utmp` 존재, 권한 `664`
- `/var/log/btmp` 존재, 권한 `600`

**취약**

- 파일 없음 또는 `last`, `lastb` 명령어 오류
