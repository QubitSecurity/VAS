## [U-35] cron 파일 소유자 및 권한 설정

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | U-35 |
| 항목명 | cron 파일 소유자 및 권한 설정 |
| 위험도 | 상 (High) |
| 범주 | 서비스 관리 |

### 점검 목적

cron 관련 파일에 부적절한 권한이 설정될 경우, 악의적인 사용자가 cron 작업에 악성 명령을 삽입하여 주기적으로 실행시킬 수 있습니다. 특히 root 권한으로 실행되는 cron 작업이 변조되면 시스템 전체가 위협을 받습니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | cron 설정 파일 소유자가 root이고 권한이 640 이하인 경우 | root만 cron 작업 추가/수정 가능 |
| 취약 | cron 설정 파일 소유자가 root가 아니거나 권한이 640을 초과하는 경우 | 일반 사용자가 cron 작업 변조 가능 |

## 베이스라인 기준

**안전(양호)**

- `/etc/crontab` 소유자 `root`, 권한 `644` 또는 `640`
- `/var/spool/cron/` 소유자 `root`, 권한 `700`
- `/etc/cron.allow` 존재 시 권한 `640` 이하

**취약**

- cron 파일에 group/other write 권한
