## [U-02] 패스워드 복잡성 설정

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | U-02 |
| 항목명 | 패스워드 복잡성 설정 |
| 위험도 | 상 (High) |
| 범주 | 계정 관리 |

### 점검 목적

단순한 패스워드는 사전 공격(Dictionary Attack) 또는 무차별 대입 공격으로 쉽게 노출됩니다. 영문 대소문자, 숫자, 특수문자의 조합을 강제함으로써 패스워드 추측 공격에 대한 저항성을 높여야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 영문 대문자, 소문자, 숫자, 특수문자 중 2종류 이상 조합 + 8자리 이상 또는 3종류 이상 조합 + 6자리 이상 | PAM pwquality 모듈로 복잡성 정책 강제 |
| 취약 | 패스워드 복잡성 정책이 설정되어 있지 않은 경우 | 단순 패스워드 사용 가능, 계정 탈취 위험 |

## 베이스라인 기준

**안전(양호)**

- `/etc/security/pwquality.conf`:
  - `minlen=8` 이상
  - `minclass=3` 이상 (또는 `dcredit=-1`, `ucredit=-1`, `lcredit=-1`, `ocredit=-1`)
- `/etc/pam.d/system-auth`: `pam_pwquality.so` 설정 포함

**취약**

- `pwquality.conf` 미설정 또는 `minclass=0`
- PAM에 `pam_pwquality.so` 미적용
