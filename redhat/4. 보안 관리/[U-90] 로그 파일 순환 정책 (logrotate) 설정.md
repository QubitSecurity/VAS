## [U-90] 로그 파일 순환 정책 (logrotate) 설정

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | U-90 |
| 항목명 | 로그 파일 순환 정책 (logrotate) 설정 |
| 위험도 | 하 (Low) |
| 범주 | 보안 관리 |

### 점검 목적

로그 파일이 순환(rotate)되지 않으면 디스크가 로그로 가득 차 서비스 장애가 발생하거나, 오래된 로그가 덮어씌워져 침해 사고 분석이 불가능해집니다. 조직 정책에 맞는 보존 기간과 압축 정책을 logrotate에 설정해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | logrotate가 설정되어 있고 적절한 보존 기간(90일 이상)이 적용된 경우 | 로그 무기한 누적 방지, 장기 보존 가능 |
| 취약 | logrotate 미설정 또는 보존 기간이 지나치게 짧은 경우 | 디스크 고갈 또는 로그 조기 삭제 |

## 베이스라인 기준

**안전(양호)**

```
weekly
rotate 13         # 13주 = 약 91일
compress
missingok
notifempty
dateext
```

**취약**

- `rotate 4` (4주 = 28일만 보존)
- logrotate 미설치 또는 미설정
- compress 없이 원본 로그 무제한 누적
