요청하신 형식에 맞춰 **WD111 웹 브라우저 계정/패스워드 저장 비활성화** 항목을 정리해 드립니다.

---

### **WD111 웹 브라우저 계정/패스워드 저장 비활성화**

**WD111 (WD111DisableBrowserPasswordStorage)** 항목은 **"웹 브라우저의 암호 관리자 기능 및 자동 로그인 사용 제한"**을 의미합니다.

한마디로 요약하면 **"해커가 가장 선호하는 탈취 대상인 '브라우저 저장 암호'를 원천적으로 생성하지 못하게 하여, PC가 악성코드에 감염되더라도 2차 피해(계정 탈취)가 발생하는 것을 막으라"**는 뜻입니다.

---

### 1. 핵심 의미: "브라우저 암호 저장소의 무력화"

대부분의 현대적 브라우저(Edge, Chrome 등)는 사용자 편의를 위해 비밀번호를 저장하고 자동으로 채워주는 기능을 제공합니다. 하지만 이 저장소는 보안상 매우 취약한 지점이 될 수 있습니다.

* **취약한 상태 (암호 저장 허용):**
* 사용자가 웹사이트 로그인 시 브라우저가 "비밀번호를 저장하시겠습니까?"라고 묻고, 저장된 암호는 브라우저 내부 DB에 보관됩니다.
* 인포스틸러(InfoStealer) 계열의 악성코드에 감염될 경우, 브라우저에 저장된 모든 사이트의 아이디와 패스워드가 순식간에 탈취됩니다.


* **안전한 상태 (기능 비활성화):**
* 브라우저가 암호 저장 여부를 묻지 않으며, 기존에 저장된 암호가 있더라도 사용하지 못하도록 정책적으로 제어됩니다.
* 모든 로그인은 사용자가 직접 입력하거나, 별도의 보안이 강화된 전용 패스워드 관리 프로그램을 통해서만 이루어집니다.



---

### 2. 점검 목적 (Why?)

"브라우저가 암호를 암호화해서 저장하는데도 위험한가요?"

네, 사용자 세션이 활성화된 상태에서는 악성코드가 브라우저의 복호화 키를 이용하여 저장된 암호를 평문(Plaintext)으로 쉽게 뽑아낼 수 있기 때문입니다.

1. **인포스틸러(InfoStealer) 대응:**
* 최근 유행하는 악성코드들은 감염 즉시 브라우저의 `Login Data` 파일을 분석하여 외부 서버로 전송합니다. 이를 방지하기 위해 저장 데이터 자체를 없애야 합니다.


2. **공용/공유 단말기 보안:**
* 여러 사람이 사용하는 관리용 서버나 공용 PC에서 이전 사용자의 계정 정보가 남아서 발생하는 무단 접속 사고를 방지합니다.


3. **계정 동기화에 따른 확산 방지:**
* 브라우저 계정 동기화 기능이 켜져 있을 경우, 사내 PC의 암호가 개인 기기로 동기화되어 보안 통제권을 벗어나는 것을 막습니다.



---

### 3. 점검 기준 (Pass/Fail)

보안 가이드라인에 따른 판단 기준입니다.

* **양호 (Safe):**
* 그룹 정책 또는 설정에서 "암호 관리자에 암호 저장 허용" 설정이 **"사용 안 함(Disabled)"**으로 강제 적용된 경우


* **취약 (Vulnerable):**
* 사용자가 브라우저 설정에서 "비밀번호 저장 여부 확인"을 켜두거나, 이미 다수의 암호가 브라우저에 저장되어 있는 경우



---

### 4. 설정 확인 및 조치 방법 (그룹 정책 관리자)

개별 브라우저 설정보다는 그룹 정책(GPO)을 통한 일괄 차단을 권장합니다.

1. **실행:** `Win + R` > `gpedit.msc` 입력
2. **경로 이동:**
* **Edge:** `컴퓨터 구성` > `관리 템플릿` > `Microsoft Edge` > `암호 관리자 및 보호`
* **Chrome:** `컴퓨터 구성` > `관리 템플릿` > `Google Chrome` > `암호 관리자`


3. **정책 찾기 및 조치:**
* **"암호 관리자에 암호 저장 허용(Enable saving passwords to the password manager)"** 항목을 찾아 **[사용 안 함(Disabled)]**으로 설정합니다.
* (선택 사항) **"비밀번호가 없는 자동 로그인 허용"** 항목도 **[사용 안 함(Disabled)]**으로 설정합니다.


4. **확인:** 브라우저 설정 메뉴의 '비밀번호' 섹션에 "조직에서 관리함" 메시지가 뜨며 설정이 비활성화되었는지 확인합니다.

### 💡 참고: 대안 제시

암호 저장 기능을 끄면 사용자들의 반발이 클 수 있습니다. 보안 강화를 위해 다음의 대안을 함께 검토하는 것이 좋습니다.

* **사내 전용 패스워드 매니저:** 마스터 패스워드와 2단계 인증(2FA)이 결합된 전문 솔루션 도입
* **SSO(Single Sign-On):** 한 번의 인증으로 사내 모든 서비스에 접근하게 하여 개별 사이트 암호 저장 필요성을 제거
