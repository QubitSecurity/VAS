KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드에 있는 **W-19 (W19RemoveIISVirtualDirectories)** 항목은 **"IIS 가상 디렉터리 삭제"**를 의미합니다.

*(참고: 가이드 버전에 따라 W-16과 내용이 비슷하거나 통합되어 보일 수 있으나, W-19는 **"기본 설치된 불필요한 가상 디렉터리의 제거"**에 더 초점을 맞춥니다.)*

한마디로 요약하면 **"IIS를 설치할 때 자동으로 만들어진 'Scripts', 'MSADC' 같은 가상 디렉터리들은 해킹의 통로가 되니, 안 쓴다면 무조건 지워라"**는 뜻입니다.

상세한 의미와 삭제 대상, 보안 목적은 다음과 같습니다.

---

### 1. 핵심 의미: "기본 제공품의 위험성"

**가상 디렉터리(Virtual Directory)**란, 실제 폴더 위치와 상관없이 웹 주소(URL)를 통해 접근할 수 있도록 연결해 주는 기능입니다.
문제는 IIS 옛날 버전이나 기본 설치 시, 관리자가 만들지도 않은 가상 디렉터리들이 자동으로 생성된다는 점입니다.

* **삭제 대상 (블랙리스트):**
* `Scripts`: 스크립트 실행용 폴더 (가장 위험)
* `MSADC`: 데이터베이스 연결 관련 (RDS 취약점)
* `IISAdmin`: 원격 관리 도구
* `IISSamples`: 예제 파일 모음
* `Printers`: 웹 프린팅 기능
* `_vti_bin`: 프론트페이지 서버 확장 (거의 안 씀)



### 2. 점검 목적 (Why?)

"안 쓰는 폴더 좀 있다고 무슨 일이 생기나요?"

이 폴더들은 해커들에게 **"공격 연습장"**이자 **"뒷문"**입니다.

1. **알려진 취약점 (Known Vulnerabilities):**
* `Scripts`나 `MSADC` 같은 기본 디렉터리에는 과거부터 수많은 보안 허점(버퍼 오버플로우 등)이 발견되었습니다.
* 해커들은 서버를 발견하면 무조건 `http://사이트/Scripts/..` 주소부터 입력해 봅니다.


2. **시스템 명령 실행:**
* 특히 `Scripts` 디렉터리는 실행 권한이 있는 경우가 많아, 해커가 악성 파일을 올리고 실행하거나 시스템 명령어를 호출하는 통로로 악용됩니다.



### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 판단 기준은 다음과 같습니다.

* **양호 (Safe):**
* `Scripts`, `MSADC`, `IISHelp`, `IISAdmin`, `Printers`, `_vti_bin` 등의 불필요한 가상 디렉터리가 **모두 제거된 경우**


* **취약 (Vulnerable):**
* 위의 기본 가상 디렉터리가 하나라도 존재하며, 사용하지 않는데도 방치된 경우



### 4. 설정 확인 및 조치 방법 (IIS 관리자)

이 작업은 **IIS 관리자**에서 눈으로 확인하고 삭제(제거)하면 됩니다.

1. **IIS 관리자 실행:** `Win + R` > `inetmgr` 입력
2. **사이트 트리 확장:** 왼쪽 메뉴에서 웹 사이트 목록(Default Web Site 등)을 펼칩니다.
3. **아이콘 확인:**
* 일반 폴더 아이콘과 달리, **지구본이나 화살표가 붙은 폴더** 아이콘을 찾습니다.


4. **목록 대조 및 삭제:**
* 다음 이름이 보이면 우클릭 후 **[제거(Remove)]**를 누릅니다.
* **Scripts**, **MSADC**, **IISAdmin**, **IISSamples**, **Printers**, **_vti_bin**
* *(주의: 실제 파일을 지우는 게 아니라, 연결(Link)만 끊는 것이므로 안전합니다. 만약 실제로 사용하는 기능이라면 지우지 않습니다.)*



### 💡 W-16(링크 사용 금지)과 W-19(가상 디렉터리 삭제)의 차이점

두 항목은 거의 같은 내용을 다루지만 미묘한 차이가 있습니다.

* **W-16:** "시스템 중요 폴더(`System32` 등)로 연결되는 링크를 만들지 마라." (정책적 관점)
* **W-19:** "기본적으로 깔린 쓰레기 폴더(`Scripts`, `MSADC` 등)를 청소해라." (청소 관점)

**결론적으로, 두 항목 모두 불필요한 가상 디렉터리를 지우면 동시에 해결됩니다.**

### 요약

> **W-19 항목은 "IIS 설치 시 딸려온 '기본 가상 디렉터리'들은 해킹 맛집이니 싹 치워라"는 뜻입니다.**

요즘 웹 환경에서는 이런 기본 디렉터리를 쓸 일이 99% 없으므로, 과감하게 지우는 것이 보안상 이득입니다.

혹시 **W-20 (IIS 웹 서비스 정보 숨김)**이나 로그 설정 등 다른 IIS 보안 항목도 궁금하신가요?
