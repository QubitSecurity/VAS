## [W-18] IIS DB 연결 취약점 점검

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-18 |
| 항목명 | IIS DB 연결 취약점 점검 |
| 위험도 | 상 (High) |
| 범주 | IIS 관리 |

### 점검 목적
데이터베이스 연결 정보(Connection String)가 웹 루트 내 파일에 평문으로 저장된 경우, 디렉토리 리스팅이나 파일 다운로드 취약점 등을 통해 DB 접속 정보가 노출될 수 있습니다. 연결 정보는 암호화하여 저장하거나 웹 루트 외부에 보관해야 합니다.

### 점검 기준
| 구분    | 기준                                             | 설명                                       |
| ----- | ---------------------------------------------- | ---------------------------------------- |
| 안전    | DB 연결 정보가 암호화되거나 웹 루트 외부에 저장된 경우               | 파일 노출 시에도 DB 접속 정보가 즉시 탈취되지 않는다          |
| 취약    | DB 연결 정보(ID, PW, 서버주소)가 웹 루트 내 파일에 평문으로 저장된 경우 | 파일 노출 시 DB 접속 정보가 즉시 공격자에게 탈취될 수 있다      |


##  **베이스라인 기준**

 **안전(양호)** 
 
• **DB 연결 정보 평문 제거**
• 설정파일 권한 **600**
• IIS: `.asa/.asax` 매핑 + **요청 필터링 모두 제거**

 **취약**
 
• 설정파일에 **DB 정보(IP/아이디/비번)** 노출
• 설정파일 권한 **>600**
• IIS `.asa/.asax "허용됨"`
