## [W-57] 원격 터미널 접속 가능한 사용자 그룹 제한

### 개요
| 항목    | 내용                      |
| ----- | ----------------------- |
| 점검 코드 | W-57                    |
| 항목명   | 원격 터미널 접속 가능한 사용자 그룹 제한 |
| 위험도   | 중 (Medium)              |
| 범주    | 접근 제어                   |

### 점검 목적
RDP(Remote Desktop Protocol)를 통한 원격 터미널 접속 권한이 불필요한 사용자나 그룹에 부여된 경우, 계정 탈취 시 공격자가 원격에서 서버에 직접 접근하여 시스템을 장악할 수 있습니다. 원격 데스크톱 접근은 반드시 업무상 필요한 최소한의 계정에만 허용해야 합니다.

### 점검 기준
| 구분    | 기준                                         | 설명                                          |
| ----- | ------------------------------------------ | ------------------------------------------- |
| 안전    | 원격 데스크톱 사용자 그룹에 업무상 필요한 최소한의 계정만 포함된 경우    | 불필요한 원격 접근 경로가 차단되어 계정 탈취 시 피해 범위가 제한된다     |
| 취약    | 원격 데스크톱 사용자 그룹에 불필요한 계정이 다수 포함된 경우         | 탈취된 계정을 통해 원격에서 서버에 직접 접근하여 시스템을 장악할 수 있다   |


**베이스라인 기준**
──────────────────────────
**양호** 
• "Remote Desktop Users" 그룹에 **관리자(Administrator) 외 별도 계정** 존재
• 원격 접속 허용 설정에서 **특정 사용자/그룹만 지정**
• Administrators 그룹은 기본 포함 **허용**

**취약**
• "Remote Desktop Users" 그룹이 **비어있음**
• **관리자 계정만으로 원격 접속** 운영
• Everyone/Guests 등 **불필요 계정 포함**
──────────────────────────

| 상태      | **정확한 조건**                                                                              | **점검 방법**                                                        |
| ------- | --------------------------------------------------------------------------------------- | ---------------------------------------------------------------- |
| ** 양호** | `lusrmgr.msc` → **Remote Desktop Users** 그룹에  <br>`Administrator` + **최소 1개의 별도 계정** 존재 | `net localgroup "Remote Desktop Users"` 명령어  <br>**2개 이상 계정 확인** |
| ** 취약** | **Remote Desktop Users 그룹이 Administrator만 존재**  <br>또는 **그룹 자체가 비어있음**                  | 그룹 멤버                                                            |
