## [W-34] 로그의 정기적 검토 및 보고

### 개요
| 항목    | 내용              |
| ----- | --------------- |
| 점검 코드 | W-34            |
| 항목명   | 로그의 정기적 검토 및 보고 |
| 위험도   | 중 (Middle)      |
| 범주    | 로그 관리           |

### 점검 목적
시스템 로그가 수집되더라도 정기적으로 검토되지 않으면 침해 사고 발생 시 사후 분석이 어렵고, 진행 중인 공격을 탐지하지 못할 수 있습니다. 로그 검토 절차와 보고 체계가 수립되어 있어야 이상 징후를 조기에 발견하고 대응할 수 있습니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 로그 정기 검토 절차가 수립되고 실제로 이행되는 경우 | 이상 징후를 조기에 발견하고 침해 사고에 신속히 대응할 수 있다 |
| 취약 | 로그 검토 절차가 없거나 형식적으로만 운영되는 경우 | 침해 사고가 발생해도 장기간 인지하지 못할 위험이 있다 |
| 확인 필요 | 로그 수집 여부는 확인되나 검토 절차 이행 여부를 자동으로 판단할 수 없는 경우 | 담당자가 로그 검토 이력 및 보고서를 직접 확인해야 한다 |
