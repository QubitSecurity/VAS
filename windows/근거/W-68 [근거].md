## [W-68] 예약된 작업에 의심스러운 명령이 등록되어 있는지 점검

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-68 |
| 항목명 | 예약된 작업에 의심스러운 명령이 등록되어 있는지 점검 |
| 위험도 | 중 (Medium) |
| 범주 | 로그 및 감사 |

### 점검 목적
Task Scheduler를 통해 등록된 작업은 시스템 재부팅 시 자동 실행되므로, 공격자가 악성 스크립트나 명령을 예약 작업으로 등록하면 지속적인 권한 상승 또는 데이터 유출을 수행할 수 있습니다. 정기적으로 예약 작업을 검토하여 의심스러운 작업이 등록되어 있는지 확인해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 모든 예약 작업이 업무상 정당한 작업만 등록되어 있는 경우 | 악성 명령을 통한 지속적 권한 상승이나 데이터 유출 경로가 없다 |
| 취약 | 예약 작업에 의심스러운 명령어, 외부 URL 호출, PowerShell 다운로드 등이 포함된 경우 | 재부팅 시 자동 실행되는 악성 명령으로 시스템이 장악될 수 있다 |
| 확인 필요 | 예약 작업 목록은 조회되나 각 작업의 정당성을 자동으로 판단할 수 없는 경우 | 담당자가 작업 명령어 및 실행 파일을 직접 검토해야 한다 |


**베이스라인 기준**
──────────────────────────
✅ **양호** 
• **작업 스케줄러**에 등록된 작업 중 **의심스러운 명령 없음**
• 모든 작업이 **업무상 정당한 프로그램** 실행

⚠️ **취약**
• 작업 명령줄에 **의심스러운 키워드** 존재:
  • `cmd.exe /c`
  • `powershell.exe`
  • `net user`, `net localgroup`
  • `whoami`, `systeminfo`
  • `certutil`, `bitsadmin`
  • `.bat`, `.vbs`, `.js`, `.ps1`
──────────────────────────
