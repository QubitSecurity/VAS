KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드에 있는 **W-28 (W28ConfigureFTPAccessControl)** 항목은 **"FTP 접근 제어 설정"**을 의미합니다.

한마디로 요약하면 **"FTP 서버에 접속할 수 있는 컴퓨터(IP 주소)를 제한해서, 허가받지 않은 PC는 접속 시도조차 못 하게 막아라"**는 뜻입니다.

상세한 의미와 보안 목적, 설정 방법은 다음과 같습니다.

---

### 1. 핵심 의미: "VIP 명단 확인 (IP 필터링)"

보통 FTP는 아이디와 비밀번호만 맞으면 누구나 접속할 수 있습니다. 하지만 이 설정을 적용하면 **"접속 위치(IP)"**를 먼저 검사합니다.

* **설정 안 함 (취약):** 전 세계 어디서든, 누구든지 우리 FTP 서버의 로그인 화면(프롬프트)까지 접근할 수 있습니다. (해커가 비밀번호 대입 공격을 시도할 기회를 줌)
* **설정 함 (양호):**
* **기본 정책:** "모든 접속 거부 (Deny All)"
* **예외 허용:** "관리자 PC IP(예: 192.168.1.10)만 허용"
*  해커가 자기 집에서 접속을 시도하면, 아이디를 묻지도 않고 **"연결이 거부되었습니다"**라며 즉시 끊어버립니다.



### 2. 점검 목적 (Why?)

"비밀번호가 있는데 굳이 IP까지 막아야 하나요?"

FTP는 보안상 매우 취약한 프로토콜이기 때문에, **이중 잠금장치**가 필수입니다.

1. **무차별 대입 공격(Brute Force) 원천 차단:**
* 해커가 자동화 도구로 1초에 수백 번씩 비밀번호를 맞추려고 시도할 때, IP 제한이 걸려있으면 아예 서버 문조차 두드릴 수 없습니다.


2. **취약점 공격 방지:**
* FTP 서비스 자체(IIS, Serv-U 등)에 보안 버그(Zero-day)가 있더라도, 접속 가능한 IP를 제한해두면 해커가 접근할 수 없어 공격을 당하지 않습니다.



### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 판단 기준은 다음과 같습니다.

* **양호 (Safe):**
* FTP 서비스에 대해 **접근 허용 IP 대역(관리자 PC 등)**이 별도로 설정되어 있고, 그 외 IP는 차단된 경우


* **취약 (Vulnerable):**
* IP 접근 제어 설정이 없어서, 모든 IP(Any/Everyone)에서 접속 가능한 경우



### 4. 설정 확인 및 조치 방법 (IIS 관리자)

Windows Server(IIS FTP)에서 IP 제한을 거는 방법입니다.

1. **IIS 관리자 실행:** `inetmgr` 입력
2. **FTP 사이트 선택:** 설정할 FTP 사이트 클릭
3. **FTP IPv4 주소 및 도메인 제한 (FTP IPv4 Address and Domain Restrictions) 더블 클릭:**
* 중앙의 기능 화면(FTP 섹션)에 있습니다.


4. **조치 (화이트리스트 방식):**
* **1단계 (모두 차단):** 우측 작업 패널의 **[기능 설정 편집(Edit Feature Settings)]**을 눌러 **"지정되지 않은 클라이언트에 대한 액세스: 거부(Deny)"**로 설정합니다.
* **2단계 (나만 허용):** 우측 작업 패널의 **[허용 항목 추가(Add Allow Entry)]**를 눌러, 접속을 허용할 관리자 PC의 IP 주소(예: `10.10.10.5`)를 입력합니다.



### 💡 주의사항: 공인 IP vs 사설 IP

* 외부(재택, 카페 등)에서 FTP에 접속해야 한다면, 해당 장소의 **공인 IP(Public IP)**를 등록해야 합니다.
* 만약 유동 IP(계속 바뀌는 IP)를 사용하는 환경이라면, IP 제한을 걸기가 까다로울 수 있습니다. 이 경우 VPN을 통해 내부망에 들어온 후 접속하게 하거나, 특정 대역(Range)을 허용하는 방법을 고려해야 합니다.

### 요약

> **W-28 항목은 "FTP 서버는 아무나 들어오면 안 되니, '초대장(허용된 IP)'을 가진 PC만 문을 열어줘라"는 뜻입니다.**

단순한 비밀번호 인증을 넘어, **네트워크 차원에서의 접근 통제**를 수행하여 보안성을 극대화하는 필수 조치입니다.
