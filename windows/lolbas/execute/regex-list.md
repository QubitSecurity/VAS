# LOLBAS Execute (168) 탐지용 정규식 정책표
작성일: 2026-03-14

이 문서는 앞서 정리한 **사용자 정의 Execute 그룹 168개**를 기준으로, **CLI 출력/Process CommandLine 문자열 기반 탐지 정책**을 설계하기 위한 운영용 정리본입니다.

> **중요:** 여기의 `Execute (168)`은 LOLBAS 원본의 `Execute`만을 뜻하지 않습니다. 앞서 재분류한 운영용 Execute 버킷을 그대로 사용했기 때문에, 원본의 `Execute`, `AWL Bypass`, `UAC Bypass`, `Compile`, 일부 `Tamper/Conceal` 계열까지 포함됩니다. 따라서 일부 도구는 **직접 실행형**, 일부는 **로더/프록시/인젝션/스테이징형**, 일부는 **CLI 단독으로는 약하고 child/registry/module correlation이 필요한 간접형**입니다.

## 설계 원칙
- 기본 전제: **명령행 문자열에서 실행파일명 + 핵심 옵션 + 파일/URL/entrypoint/remote indicator**를 탐지한다.
- 정규식은 **대소문자 무시 `(?i)`** 기준이다.
- **직접 실행형**과 **간접형(UAC/registry hijack, path hijack, rename proxy, child-only behavior)**을 같은 정확도로 취급하지 않는다.
- DLL/INF/COM/CPL/ClickOnce/Script/XSL/PowerShell 계열은 **호스트 바이너리와 대상 파일명/entrypoint를 함께** 보는 것이 좋다.
- 일부 희귀 도구는 CLI만으로 확정이 어려우므로 **Parent-Child / Module Load / Registry / Working Directory / Network / File Create**와 결합해야 한다.
