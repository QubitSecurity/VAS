KISA(한국인터넷진흥원)의 주요정보통신기반시설 취약점 분석·평가 가이드에 있는 **W-38 (W38ConfigureScreenSaverSettings)** 항목은 **"화면보호기 설정"**을 의미합니다.

이 항목의 핵심 의미는 단순히 화면을 보호하는 것이 아니라, **"관리자가 자리를 비웠을 때 자동으로 컴퓨터를 잠가서(Lock), 타인의 무단 접근을 차단하라"**는 것입니다.

상세한 내용과 점검 목적은 다음과 같습니다.

---

### 1. 핵심 의미: 자동 잠금 장치

서버 관리자가 작업을 하다가 급한 용무로 자리를 비웠을 때, 로그아웃을 하지 않았다면 어떤 일이 벌어질까요? 지나가던 사람이나 악의적인 내부자가 열려 있는 화면을 통해 시스템을 조작할 수 있습니다.

이 항목은 이를 방지하기 위해 다음 **3가지 조건**을 강제하도록 요구합니다.

1. **화면보호기 활성화:** 일정 시간 사용이 없으면 기능이 켜져야 합니다.
2. **대기 시간 (Timeout):** 너무 늦게 켜지면 안 됩니다. (보통 **10분** 이내 권장)
3. **암호 사용 (Resume Password):** 화면보호기가 해제될 때, **반드시 다시 로그인을 하도록(암호 입력)** 설정되어 있어야 합니다.

### 2. 점검 목적 (Why?)

* **비인가자 접근 통제:** 물리적으로 서버실에 접근 가능한 인원이나, 원격 접속(RDP) 상태에서 자리를 비운 사이 제3자가 서버를 조작하는 것을 방지합니다.
* **세션 보호:** 관리자 권한이 있는 세션이 방치되는 것을 막습니다.

### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 일반적인 판단 기준은 다음과 같습니다.

* **양호 (Safe):**
* 화면보호기가 설정되어 있고,
* 대기 시간이 **10분 이하**이며,
* **"다시 시작할 때 로그온 화면 표시"** 기능이 활성화된 경우


* **취약 (Vulnerable):**
* 화면보호기가 꺼져 있거나,
* 대기 시간이 10분을 초과하거나,
* 암호를 묻지 않고 바로 바탕화면으로 돌아가는 경우



### 4. 설정 확인 및 조치 방법 (Windows Server 기준)

이 설정은 주로 **로컬 그룹 정책 편집기(Local Group Policy Editor)**를 통해 제어하는 것이 가장 확실합니다.

1. **실행 창 열기:** `Win + R` 키를 누르고 `gpedit.msc` 입력
2. **경로 이동:**
* `사용자 구성(User Configuration)` > `관리 템플릿(Administrative Templates)` > `제어판(Control Panel)` > `개인 설정(Personalization)`


3. **설정 변경 (아래 3가지를 모두 설정해야 함):**
* **화면 보호기 사용 (Enable Screen Saver):** `사용(Enabled)`
* **화면 보호기 시간 제한 (Screen Saver Timeout):** `사용(Enabled)` → 초 단위 입력 (예: **600초** = 10분)
* **화면 보호기 암호로 보호 (Password protect the screen saver):** `사용(Enabled)`



### 요약

> **W-38 항목은 "자리를 비울 때(10분) 자동으로 컴퓨터를 잠그고(암호), 남이 못 건드르게 하라"는 뜻입니다.**

단순히 모니터 번인(Burn-in)을 막는 옛날 화면보호기의 용도가 아니라, **자동 세션 잠금(Auto Lock)**을 위한 보안 조치라고 이해하시면 가장 정확합니다.

현재 점검하시는 서버가 화면보호기 설정이 되어 있지 않거나 시간이 너무 길게(예: 60분) 설정되어 있다면 '취약'으로 탐지됩니다. 도움이 되셨나요?
