KISA(한국인터넷진흥원)의 주요정보통신기반시설 취약점 분석·평가 가이드에 있는 **W-53 (W53AllowLocalLogon)** 항목은 **"로컬 로그온 허용"** 정책 설정을 의미합니다.

한마디로 요약하면 **"서버 컴퓨터 앞에 직접 앉아서(또는 원격 데스크톱으로) 키보드를 두드리며 로그인할 수 있는 권한을 누구에게 줄 것인가?"**를 결정하는 항목입니다.

이 항목의 핵심 의미와 보안 목적은 다음과 같습니다.

---

### 1. 핵심 의미: "서버실 출입증"

IT 환경에서 '로그인'은 크게 두 가지로 나뉩니다.

1. **네트워크 로그인:** 파일 공유나 웹사이트 접속처럼, 서비스를 이용하기 위해 네트워크를 통해 접근하는 것.
2. **로컬 로그온 (이 항목):** 키보드와 모니터가 연결된 콘솔에서 직접 로그인하거나, 원격 데스크톱(RDP)처럼 윈도우 화면을 보며 로그인하는 것.

**W-53**은 이 중에서 **'로컬 로그온'** 권한을 제어합니다.

* **취약한 상태:** `Guests`나 `Users` 같은 일반 사용자 그룹이 포함되어 있어, 권한 없는 사용자가 서버 내부에 진입을 시도할 수 있는 상태.
* **안전한 상태:** `Administrators` 같은 **관리자 그룹만** 이 권한을 가지고 있어, 일반 사용자는 서버 콘솔에 접근조차 못 하는 상태.

### 2. 점검 목적 (Why?)

서버는 일반 PC(클라이언트)가 아닙니다. 일반 사용자는 서버가 제공하는 '서비스(DB, 웹)'만 이용하면 되지, 서버의 운영체제 화면에 들어올 이유가 없습니다.

1. **비인가자 접근 통제:** 불필요한 사용자가 서버 내부(Desktop)에 진입하면, 로컬 취약점을 이용해 권한 상승(해킹)을 시도하거나 시스템 설정을 건드릴 수 있습니다.
2. **공격 표면 최소화:** `Guests` 그룹 등이 포함되어 있으면, 해커가 획득한 낮은 권한의 계정으로도 서버에 로그온하여 내부 정찰을 할 수 있게 됩니다.

### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 판단 기준은 다음과 같습니다.

* **양호 (Safe):** 로컬 로그온 허용 그룹에 **`Administrators`**만 존재하거나, 불필요한 그룹(Guests 등)이 제거된 경우
* **취약 (Vulnerable):** `Guests`, `Everyone` 등 불필요한 일반 사용자 그룹이 포함된 경우

*(참고: 서버 용도에 따라 `Backup Operators` 등 특정 운영 그룹은 필요할 수 있으나, 일반적인 `Users` 그룹은 제외하는 것이 원칙입니다.)*

### 4. 설정 확인 및 조치 방법

이 설정은 **사용자 권한 할당(User Rights Assignment)** 정책에서 관리합니다.

1. **실행 창 열기:** `Win + R` 키를 누르고 `secpol.msc` 입력
2. **경로 이동:**
* `로컬 정책(Local Policies)` > `사용자 권한 할당(User Rights Assignment)`


3. **정책 찾기:**
* **"로컬 로그온 허용 (Allow log on locally)"** 항목을 더블 클릭합니다.


4. **조치:**
* 목록에 `Administrators` 외에 **`Guests`**, **`Everyone`**, **`Users`** 등이 있다면 선택 후 **[제거]** 버튼을 누릅니다.
* 확인을 눌러 저장합니다.



### 💡 주의사항: 원격 데스크톱(RDP) 사용자

만약 이 서버에 원격 데스크톱(RDP)으로 접속해야 하는 **'관리자가 아닌 특정 사용자'**가 있다면 주의해야 합니다.

* 원격 데스크톱 접속은 기술적으로 **'로컬 로그온'** 권한을 필요로 합니다.
* 따라서 `Remote Desktop Users` 그룹을 여기서 무조건 삭제하면, 해당 그룹 사용자들의 원격 접속이 막힐 수 있습니다.
* **결론:** `Guests`, `Everyone`은 무조건 삭제하되, 업무상 필요한 `Remote Desktop Users` 그룹은 상황에 따라 남겨둘 수 있습니다. (단, 관리자 전용 서버라면 이마저도 빼고 관리자만 남기는 것이 가장 안전합니다.)

### 요약

> **W-53 항목은 "아무나 서버 컴퓨터 켜고 로그인하지 못하게 하라"는 뜻입니다.**
> 서버는 관리자(Administrator)의 영역이므로, 일반 사용자나 손님(Guest) 계정은 로그온 권한을 뺏어야 안전합니다.
