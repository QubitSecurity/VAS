KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드에 있는 **W-62 (W62ConfigureSNMPAccessControl)** 항목은 **"SNMP 접근 제어(Access Control) 설정"**을 의미합니다.

한마디로 요약하면 **"SNMP 서비스를 사용할 때, 우리 회사의 관리 서버(NMS) IP 주소에서 오는 요청만 받아주고, 나머지 모르는 IP는 무시하라"**는 뜻입니다.

상세한 의미와 보안 목적, 설정 방법은 다음과 같습니다.

---

### 1. 핵심 의미: "VIP 명단 확인 (IP 필터링)"

W-61(비밀번호 복잡성)이 **"암호가 맞는지"** 확인하는 것이라면, W-62는 **"어디서 왔는지"** 확인하는 것입니다.

* **취약한 설정 (Any Host):**
* "모든 호스트로부터의 SNMP 패킷 받아들이기"가 체크되어 있습니다.
* 전 세계 어디서든, 해커가 비밀번호(Community String)만 알아내면 서버 정보를 털어갈 수 있습니다.


* **안전한 설정 (Specific Hosts):**
* "다음 호스트로부터의 SNMP 패킷 받아들이기"가 체크되어 있고, 관리 서버(NMS)의 IP(예: `192.168.10.5`)만 등록되어 있습니다.
* 해커가 비밀번호를 알아내더라도, IP가 다르므로 접근이 거부됩니다.



### 2. 점검 목적 (Why?)

"비밀번호(W-61)를 어렵게 했는데, 굳이 IP까지 막아야 하나요?"

네, 보안에는 **'방어 심층(Defense in Depth)'** 원칙이 적용됩니다. 비밀번호가 뚫려도 IP에서 막히도록 **이중 잠금장치**를 거는 것입니다.

1. **스니핑(Sniffing) 방어:** SNMP v1, v2c는 비밀번호가 네트워크상에 평문으로 전송됩니다. 해커가 이를 도청해서 비밀번호를 알아내더라도, 해커의 IP로는 접속할 수 없게 막아줍니다.
2. **무차별 대입 공격 차단:** 해커가 비밀번호를 맞추려고 시도할 기회조차 주지 않습니다.

### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 판단 기준은 다음과 같습니다.

* **양호 (Safe):**
* SNMP 서비스를 **사용하지 않는 경우** (W-60 조치 시 자동 양호)
* 사용하더라도 **"지정된 호스트(IP)"**에서만 접속을 허용하도록 설정한 경우


* **취약 (Vulnerable):**
* SNMP 서비스가 실행 중이고, **"모든 호스트(Any Host)"**로부터 접속을 허용하는 경우



### 4. 설정 확인 및 조치 방법 (Windows 서비스)

이 설정은 방화벽이 아니라, **SNMP 서비스 자체 설정**에서 제어합니다.

1. **서비스 속성 열기:** `Win + R` > `services.msc` > `SNMP Service` 더블 클릭
2. **보안 탭 이동:** 상단의 **[보안(Security)]** 탭 클릭
3. **호스트 제한 설정:**
* 하단의 옵션 중 **"모든 호스트로부터의 SNMP 패킷 받아들이기"** 체크를 **해제**합니다.
* **"다음 호스트로부터의 SNMP 패킷 받아들이기"**를 선택합니다.
* **[추가(Add)]** 버튼을 눌러, 실제로 모니터링을 수행하는 **관리 서버(NMS)의 IP 주소**를 입력합니다.
* *(자신의 로컬 호스트 `127.0.0.1`이나 `localhost`도 필요하다면 추가합니다.)*



### 요약

> **W-62 항목은 "SNMP는 보안이 약하니까, '아는 사람(허용된 IP)'하고만 대화하게 설정하라"는 뜻입니다.**

이로써 **SNMP 관련 3종 세트(W-60, W-61, W-62)**를 모두 확인하셨습니다.

1. **W-60:** 안 쓰면 꺼라.
2. **W-61:** 쓸 거면 비밀번호 어렵게 해라.
3. **W-62:** 쓸 거면 허용된 IP만 받아라.

이 세 가지만 지키면 SNMP를 통한 해킹 위협은 거의 완벽하게 차단됩니다.

이제 윈도우 서버 취약점 점검 항목에 대해 상당히 많이 파악하셨습니다! 혹시 **리눅스(Unix) 서버** 점검 항목이나, **DBMS(데이터베이스)** 관련 점검 항목으로 넘어가시겠습니까?
