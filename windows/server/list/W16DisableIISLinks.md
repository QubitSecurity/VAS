KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드에 있는 **W-16 (W16DisableIISLinks)** 항목은 **"IIS 링크 사용 금지"**를 의미합니다.

여기서 말하는 '링크'는 우리가 흔히 클릭하는 웹사이트 주소(하이퍼링크)를 말하는 것이 아니라, **"심볼릭 링크(Symbolic Link)"**나 **"가상 디렉터리(Virtual Directory)"**와 같이 **폴더와 폴더를 연결하는 바로가기 기능**을 의미합니다.

한마디로 요약하면 **"웹 사이트 폴더 안에, 윈도우의 중요 시스템 폴더(C드라이브 루트, System32 등)로 바로 연결되는 비밀 통로(바로가기)를 만들지 마라"**는 뜻입니다.

상세한 의미와 위험성, 조치 방법은 다음과 같습니다.

---

### 1. 핵심 의미: "위험한 지름길"

웹 서버는 원칙적으로 약속된 홈 디렉터리(예: `C:\Inetpub\wwwroot`) 안에 있는 파일만 보여줘야 합니다.

하지만 IIS의 **가상 디렉터리** 기능을 이용하면, 홈 디렉터리 밖에 있는 폴더를 마치 안에 있는 것처럼 연결할 수 있습니다.

* **취약한 상태:** 웹 사이트 하위에 `\Scripts`나 `\System`이라는 가상 디렉터리가 존재하고, 이것이 실제로는 `C:\Windows\System32` 폴더를 가리키고 있는 경우.
* **안전한 상태:** 웹 서비스에 꼭 필요한 폴더 외에, 시스템 내부로 연결되는 불필요한 가상 디렉터리가 모두 삭제된 상태.

### 2. 점검 목적 (Why?)

"편하게 쓰려고 연결해둔 건데 왜 막나요?"

이 '지름길'이 해커에게는 **고속도로**가 되기 때문입니다.

1. **시스템 명령어 실행:** 만약 `\Scripts` 폴더가 `System32` 폴더와 연결되어 있고 실행 권한이 있다면, 해커는 웹 브라우저 주소창에 `http://사이트/Scripts/cmd.exe`를 입력하여 서버의 커맨드 창을 마음대로 실행할 수 있습니다.
2. **보안 우회:** 웹 루트 디렉터리에 설정된 보안 정책을 우회하여, 접근해서는 안 되는 시스템 중요 파일에 접근할 수 있습니다.

### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 판단 기준은 다음과 같습니다.

* **양호 (Safe):**
* 불필요한 가상 디렉터리(Scripts, MSADC, IISAdmin, IISSamples 등)가 존재하지 않는 경우
* 존재하더라도 실제 시스템 경로(System32 등)와 연결되어 있지 않은 경우


* **취약 (Vulnerable):**
* 기본 설치 시 생성된 가상 디렉터리가 제거되지 않고 남아있는 경우
* 가상 디렉터리가 유추하기 쉬운 이름으로 시스템 중요 폴더를 가리키고 있는 경우



### 4. 설정 확인 및 조치 방법 (IIS 관리자)

IIS 관리 콘솔에서 불필요한 가상 디렉터리가 있는지 눈으로 확인하고 지워야 합니다.

1. **IIS 관리자 실행:** `Win + R` > `inetmgr` 입력
2. **사이트 트리 확장:** 왼쪽 메뉴에서 웹 사이트 목록을 펼칩니다.
3. **아이콘 확인:**
* 일반 폴더 아이콘이 아니라, **폴더에 지구본 모양이나 화살표가 붙어 있는 아이콘**이 '가상 디렉터리' 또는 '애플리케이션'입니다.


4. **목록 점검:**
* 다음과 같은 이름이 보이면 내용을 확인해야 합니다.
* `Scripts`, `MSADC`, `IISHelp`, `IISAdmin`, `Printers`, `_vti_bin`


5. **조치 (삭제):**
* 실제 운영 서비스에 사용하지 않는 가상 디렉터리라면, 우클릭 후 **[제거(Remove)]**를 선택합니다.
* *(주의: 이는 실제 파일을 지우는 게 아니라, 연결 고리(링크)만 끊는 것이므로 안전합니다.)*



### 요약

> **W-16 항목은 "웹 사이트 내부에 시스템 깊숙한 곳으로 직행하는 '바로가기'를 두지 마라"는 뜻입니다.**

특히 IIS를 기본 설치했을 때 자동으로 생기는 `Scripts` 같은 가상 디렉터리는 과거에 해킹 사고의 주원인이었으므로 반드시 확인하고 지워야 합니다.

혹시 **W-17 (웹 프로세스 권한)**이나 다른 웹 서버 설정에 대해서도 궁금한 점이 있으신가요?
