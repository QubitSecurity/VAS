KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드에 있는 **W-63 (W63CheckIfDNSServiceIsRunning)** 항목은 **"DNS 서비스 실행 여부 점검"**을 의미합니다.

한마디로 요약하면 **"이 서버가 DNS(도메인 이름 풀이) 전용 서버가 아니라면, DNS 서비스 기능을 아예 꺼라"**는 뜻입니다.

상세한 의미와 왜 꺼야 하는지, 어떻게 조치해야 하는지 설명해 드리겠습니다.

---

### 1. 핵심 의미: "불필요한 기능 = 보안 구멍"

모든 서버는 인터넷을 하기 위해 **DNS 클라이언트(Client)** 기능은 필요합니다. 하지만 남들의 도메인 주소를 알려주는 **DNS 서버(Server)** 기능은 특별한 경우에만 필요합니다.

* **DNS 서버가 아닌 경우 (웹/DB 서버 등):** DNS 서비스를 켜둘 이유가 전혀 없습니다. 그런데 윈도우 설치 시 실수로 설치했거나 기본으로 켜져 있다면, 이것이 바로 **취약점**입니다.
* **DNS 서버인 경우:** 당연히 켜져 있어야 합니다. (이 경우 W-63은 '점검' 단계에서 넘어가고, W-29, W-64 등 세부 보안 설정을 확인하게 됩니다.)

### 2. 점검 목적 (Why?)

"안 쓰는 기능 좀 켜져 있으면 어떤가요?"

사용하지도 않는 DNS 서비스가 켜져 있으면 해커에게 **강력한 무기**를 쥐어주는 꼴이 됩니다.

1. **DDoS 공격 악용 (DNS Amplification):**
* 해커가 우리 서버를 **"오픈 리졸버(Open Resolver)"**로 악용할 수 있습니다.
* 해커는 아주 작은 질문 패킷을 보내면서, 출발지 IP를 피해자(타겟)의 IP로 위조합니다.
* 우리 서버는 피해자에게 **엄청나게 큰 응답 패킷**을 보냅니다. (증폭 공격)
* 결국 우리 서버가 **남을 공격하는 DDoS 공격의 공범**이 됩니다.


2. **DNS 캐시 포이즈닝 (Cache Poisoning):**
* 해커가 우리 서버의 DNS 정보를 조작하여, 사용자를 가짜 사이트(피싱 사이트)로 유도할 수 있습니다.


3. **서버 자원 낭비:**
* 불필요한 DNS 요청을 처리하느라 CPU와 네트워크 대역폭을 낭비하게 됩니다.



### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 판단 기준은 **서버의 용도**에 따라 달라집니다.

* **양호 (Safe):**
* DNS 서비스를 **사용하지 않아 서비스가 중지(Stopped)**되어 있거나, 아예 **설치되지 않은** 경우
* DNS 전용 서버로서 **필요에 의해** 실행 중인 경우 (단, 보안 설정이 되어 있어야 함)


* **취약 (Vulnerable):**
* DNS 서버 용도가 아닌데도 DNS 서비스가 **"실행 중(Running)"**인 경우



### 4. 확인 및 조치 방법

이 서버가 DNS 서버가 아니라면, 다음 방법으로 서비스를 내리거나 제거해야 합니다.

#### A. 서비스 중지 (일시적 조치)

1. **실행 창 열기:** `Win + R` 누르고 `services.msc` 입력
2. **서비스 찾기:**
* **"DNS Server"** 항목을 찾습니다. (주의: **"DNS Client"**는 끄면 안 됩니다! 인터넷이 안 될 수 있습니다.)


3. **조치:**
* 더블 클릭 후 시작 유형을 **[사용 안 함]**으로 변경하고, 서비스 상태를 **[중지]**로 설정합니다.



#### B. 역할 제거 (권장 - 영구적 조치)

아예 윈도우에서 DNS 기능을 삭제하는 것이 가장 안전합니다.

1. **서버 관리자(Server Manager)** 실행
2. **관리(Manage)** > **역할 및 기능 제거(Remove Roles and Features)** 클릭
3. **"DNS 서버"** 체크 박스를 해제하고 제거 진행

### 요약

> **W-63 항목은 "DNS 서버도 아니면서 왜 53번 포트(DNS)를 열어놓고 있느냐?"라고 묻는 항목입니다.**
> 웹서버나 DB서버라면, 보안을 위해 DNS 서버 역할은 반드시 삭제해야 합니다.

혹시 현재 점검 중인 서버가 DNS 서버인가요, 아니면 다른 용도의 서버인가요? 만약 DNS 서버라면 **W-29 (영역 전송 제한)**이나 **W-64 (동적 업데이트 제한)** 같은 추가적인 DNS 보안 설정도 점검해야 합니다.
