KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드에 있는 **W-74 (W74SetIdleTimeBeforeDisconnectingSessions)** 항목은 **"세션 연결 중단 전 유휴시간 설정"**을 의미합니다.

한마디로 요약하면 **"원격 데스크톱(RDP)으로 서버에 접속해 놓고, 일정 시간 동안 마우스나 키보드를 건드리지 않으면(잠수 타면) 자동으로 연결을 끊어버려라"**는 뜻입니다.

상세한 의미와 보안 목적, 설정 방법은 다음과 같습니다.

---

### 1. 핵심 의미: "자동 로그아웃(Kick out)"

은행 웹사이트나 앱을 켜놓고 가만히 있으면 "로그인 시간이 연장되었습니다"라는 알림이 뜨면서 자동으로 튕기는 것을 보셨을 겁니다. 서버도 똑같습니다.

* **취약한 상태 (제한 없음):**
* 관리자가 원격 데스크톱으로 서버에 접속한 뒤, **로그아웃을 안 하고 그냥 창만 닫거나 켜둔 채로 퇴근/외출**을 합니다.
* 이 세션은 서버 메모리에 계속 살아있게 되며, 연결이 끊어지지 않습니다.


* **안전한 상태 (시간 제한 설정):**
* "10분 동안 아무 입력이 없으면 연결 끊기" 정책을 겁니다.
* 관리자가 자리를 비우면 서버가 알아서 연결을 끊어버립니다.



### 2. 점검 목적 (Why?)

"편하게 켜두고 작업하면 안 되나요?"

보안상 **'열려 있는 세션'**은 언제나 위험합니다.

1. **세션 하이재킹(Hijacking) 방지:**
* 관리자가 자리를 비운 사이, 공격자나 악의적인 내부자가 해당 PC에 접근하여 **이미 열려 있는 서버 원격 세션**을 통해 중요 정보를 탈취하거나 서버를 조작할 수 있습니다.


2. **리소스 낭비 방지:**
* 제대로 종료되지 않은 '유령 세션(Zombie Session)'들이 서버의 메모리와 CPU 자원을 계속 잡아먹는 것을 막습니다.


3. **중간자 공격 위험 감소:**
* 연결이 오래 유지될수록 해커가 중간에서 세션을 가로챌(Session Hijacking) 기회가 늘어납니다.



### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 판단 기준은 다음과 같습니다.

* **양호 (Safe):**
* 유휴 시간 제한 정책이 **"사용(Enabled)"**으로 설정되어 있고,
* 시간 제한이 **10분 ~ 15분 이하**로 설정된 경우 (조직 정책에 따라 다름)


* **취약 (Vulnerable):**
* 정책이 **"사용 안 함(Disabled)"** 또는 **"구성되지 않음(Not Configured)"**으로 되어 있어, 무제한 연결이 가능한 경우



### 4. 설정 확인 및 조치 방법 (그룹 정책)

이 설정은 윈도우의 **로컬 그룹 정책 편집기(gpedit.msc)**에서 설정합니다.

1. **실행:** `Win + R` > `gpedit.msc` 입력
2. **경로 이동:**
* **컴퓨터 구성** > **관리 템플릿** > **Windows 구성 요소** > **터미널 서비스(또는 원격 데스크톱 서비스)** > **원격 데스크톱 세션 호스트** > **세션 시간 제한**
* *(English: Computer Configuration > Administrative Templates > Windows Components > Remote Desktop Services > RD Session Host > Session Time Limits)*


3. **정책 선택:**
* **"활성 상태이지만 사용하지 않는 원격 데스크톱 서비스 세션에 시간 제한 설정 (Set time limit for active but idle Remote Desktop Services sessions)"** 항목을 더블 클릭합니다.


4. **조치:**
* **[사용(Enabled)]**을 선택합니다.
* 아래 옵션의 **'유휴 세션 제한'** 시간을 **10분** 또는 **15분** 등으로 설정합니다.



### 💡 주의사항: '연결 끊기' vs '세션 종료'

이 정책에는 두 가지 옵션이 더 있습니다. 헷갈리기 쉬우니 주의하세요.

* **W-74 (유휴 시간 제한):** "일단 연결만 끊어!" (프로그램은 서버에서 계속 돌아감)  다시 접속하면 작업하던 화면 그대로 뜸.
* **세션 종료 시간 제한:** "연결 끊기면 아예 강제 로그아웃 시켜!" (프로그램 다 꺼짐)  메모리 확보에는 좋지만, 작업 중이던 데이터가 날아갈 수 있음.

**W-74는 우선 "유휴 시간(Idle Time)"에 대한 제한을 거는 것이 핵심입니다.**

### 요약

> **W-74 항목은 "원격 접속해놓고 딴짓(잠수)하면, 보안을 위해 강제로 튕겨내라"는 뜻입니다.**
