KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드에 있는 **W-05 (W05DisableReversibleEncryptionForPasswordStorage)** 항목은 윈도우(Windows) 서버의 보안 설정 중 하나입니다.

한마디로 요약하면 **"비밀번호를 원래대로 복구할 수 있는(해독 가능한) 형태로 저장하지 말라"**는 뜻입니다.

이 항목의 핵심 의미와 보안상 중요한 이유는 다음과 같습니다.

---

### 1. 핵심 의미: 가역(Reversible) vs. 일방향(One-way)

이 항목을 이해하기 위해서는 비밀번호 저장 방식의 두 가지 차이를 알아야 합니다.

* **가역 암호화 (취약함 - 점검 대상):**
* 비밀번호를 암호화해서 저장하지만, **암호화 키만 있으면 다시 원래의 비밀번호(평문)로 되돌릴 수 있는 방식**입니다.
* 예: `1234`  `AbCd...`  `1234`
* **위험성:** 해커가 서버를 해킹하여 암호화된 파일과 키를 탈취하면, 모든 사용자의 비밀번호를 알아낼 수 있습니다.


* **일방향 암호화/해시 (안전함 - 권장):**
* 비밀번호를 수학적 연산(Hash)을 통해 변환하지만, **변환된 값을 다시 원래 비밀번호로 되돌릴 수는 없는 방식**입니다.
* 예: `1234`  `Xy9z...`  (X)
* **안전성:** 해커가 데이터를 훔쳐가도 원래 비밀번호가 무엇인지 알 수 없습니다. 사용자가 로그인할 때 입력한 값을 똑같이 해시화해서 비교만 합니다.



### 2. 점검 목적 (Why?)

윈도우 서버에는 **"해독 가능한 암호화를 사용하여 암호 저장(Store passwords using reversible encryption)"**이라는 정책 옵션이 있습니다.

* 이 옵션이 **'사용(Enabled)'**으로 되어 있으면, 운영체제는 사용자의 비밀번호를 복호화가 가능한 형태로 디스크에 저장합니다.
* 이는 주로 아주 오래된 애플리케이션이나 특정 인증 프로토콜(CHAP, IIS의 다이제스트 인증 등)을 지원하기 위해 존재했던 기능입니다.
* 하지만 현대적인 보안 환경에서는 **치명적인 보안 홀**이 될 수 있으므로, 특수한 경우가 아니라면 반드시 꺼야 합니다.

### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 판단 기준은 다음과 같습니다.

* **양호 (Safe):** 해당 정책이 **"사용 안 함(Disabled)"**으로 설정된 경우
* **취약 (Vulnerable):** 해당 정책이 **"사용(Enabled)"**으로 설정된 경우

### 4. 설정 확인 및 조치 방법

윈도우 서버에서 이 설정을 확인하거나 변경하는 방법은 다음과 같습니다.

1. **실행 창 열기:** `Win + R` 키를 누르고 `secpol.msc` (로컬 보안 정책) 입력 후 실행
2. **경로 이동:**
* `보안 설정(Security Settings)` > `계정 정책(Account Policies)` > `암호 정책(Password Policy)`


3. **정책 확인:**
* **"해독 가능한 암호화를 사용하여 암호 저장(Store passwords using reversible encryption)"** 항목 찾기


4. **조치:**
* 해당 항목을 더블 클릭하여 **[사용 안 함]**으로 체크하고 확인합니다.



### 요약

> **W-05 항목은 "비밀번호가 털려도 해커가 원래 암호를 알 수 없게 하라"는 뜻입니다.** 따라서 특별한 레거시(구형) 시스템을 쓰는 게 아니라면, 이 기능은 무조건 **꺼져 있어야(Disabled)** 안전합니다.

혹시 현재 서버 점검 중 이 항목이 '취약'으로 나오셨나요? 그렇다면 바로 설정을 변경하는 절차를 안내해 드릴까요?
