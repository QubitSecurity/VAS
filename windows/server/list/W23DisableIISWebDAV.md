KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드에 있는 **W-23 (W23DisableIISWebDAV)** 항목은 **"IIS WebDAV 비활성화"**를 의미합니다.

한마디로 요약하면 **"웹 서버를 마치 'USB 메모리'나 '네트워크 드라이브'처럼 파일 쓰기/수정/삭제가 가능하게 만드는 기능을 꺼라"**는 뜻입니다.

상세한 의미와 보안 위험성, 조치 방법은 다음과 같습니다.

---

### 1. 핵심 의미: "읽기 전용 vs. 읽기/쓰기"

일반적인 웹 서핑(HTTP)은 서버에 있는 정보를 **'가져오는(GET)'** 행위가 주를 이룹니다.
하지만 **WebDAV (Web Distributed Authoring and Versioning)**는 여기서 한 발 더 나아가, 사용자가 웹 서버에 파일을 **'올리고(PUT)', '지우고(DELETE)', '수정하는'** 작업을 할 수 있게 해주는 확장 프로토콜입니다.

* **WebDAV 켜짐 (취약):** 누군가(권한이 탈취된 사용자)가 웹 브라우저나 탐색기를 통해 서버의 파일을 직접 수정하거나 삭제할 수 있습니다.
* **WebDAV 꺼짐 (양호):** 웹 서버는 오직 정보 제공(View)만 수행하며, 파일 조작은 불가능합니다.

### 2. 점검 목적 (Why?)

"편리한 기능인데 왜 끄라고 하나요?"

이 기능이 켜져 있으면 공격자에게 **너무 강력한 권한(파일 조작)**을 줄 위험이 있기 때문입니다.

1. **웹 변조 (Defacement):** 해커가 메인 페이지(`index.html`)를 덮어쓰거나 삭제하여 사이트를 마비시킬 수 있습니다.
2. **악성 파일 업로드:** 해커가 웹 쉘(Web Shell)이나 악성코드 파일을 서버에 직접 업로드(`PUT`)하여 시스템을 장악할 수 있습니다.
3. **인증 우회 취약점:** 과거 IIS의 WebDAV 컴포넌트에서 인증을 우회하거나 버퍼 오버플로우를 일으키는 심각한 보안 취약점이 다수 발견된 바 있습니다.

### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 판단 기준은 다음과 같습니다.

* **양호 (Safe):** WebDAV 기능이 **설치되어 있지 않거나**, 설치되었더라도 **비활성화(Disable)** 또는 **차단(Deny)** 설정된 경우
* **취약 (Vulnerable):** WebDAV 기능이 활성화되어 있고, 적절한 접근 통제가 이루어지지 않은 경우

### 4. 설정 확인 및 조치 방법

WebDAV를 끄는 방법은 두 가지가 있습니다. 아예 삭제하는 것이 가장 좋습니다.

#### 방법 A. 역할 제거 (가장 권장 - 영구 삭제)

서버에서 WebDAV 기능 자체를 들어내는 방법입니다.

1. **서버 관리자(Server Manager)** 실행
2. **관리(Manage)** > **역할 및 기능 제거(Remove Roles and Features)**
3. **웹 서버(IIS)** > **웹 서버** > **일반적인 HTTP 기능**
4. **"WebDAV 게시(WebDAV Publishing)"** 체크 해제 후 제거 진행

#### 방법 B. 기능 비활성화 (IIS 관리자)

삭제하기 어렵다면 설정에서 기능을 끕니다.

1. **IIS 관리자 실행:** `inetmgr` 입력
2. **사이트 선택:** 해당 웹 사이트 클릭
3. **WebDAV 저작 규칙(WebDAV Authoring Rules) 더블 클릭:**
* *(이 아이콘이 없다면 WebDAV가 설치되지 않은 것이므로 안전합니다.)*


4. **조치:**
* 우측 작업 패널에서 **[WebDAV 사용 안 함(Disable WebDAV)]**을 클릭합니다.
* 또는 기존에 등록된 모든 허용 규칙을 삭제합니다.



### 💡 주의사항: SharePoint나 Exchange Server

만약 현재 운영 중인 서버가 **MS SharePoint**, **Exchange Server(OWA)**, 또는 **SVN(Subversion)** 등을 구동하는 서버라면, 이들은 내부적으로 WebDAV를 필수적으로 사용합니다.

* **이런 경우:** WebDAV를 끄면 안 됩니다.
* **대안:** WebDAV를 켜두되, **IP 접근 제어**나 **강력한 인증(Authentication)**을 걸어서 아무나 접근하지 못하게 설정해야 '양호'로 인정받을 수 있습니다.

### 요약

> **W-23 항목은 "웹 서버를 '공용 클라우드'처럼 쓰지 않으려면 WebDAV를 꺼라"는 뜻입니다.**

단순 홈페이지 운영이 목적이라면, WebDAV는 해커에게 파일 업로드/삭제 권한만 주는 위험한 기능일 뿐입니다. 반드시 삭제하거나 비활성화하세요.
