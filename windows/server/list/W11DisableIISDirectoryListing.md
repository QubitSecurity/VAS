KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드에 있는 **W-11 (W11DisableIISDirectoryListing)** 항목은 **"IIS 디렉터리 리스팅(목록 조회) 기능 끄기"**를 의미합니다.

한마디로 요약하면 **"웹 사이트의 특정 폴더 주소로 접속했을 때, 그 안에 들어있는 파일 목록이 탐색기처럼 훤히 다 보이는 현상을 막아라"**는 뜻입니다.

상세한 의미와 보안 위협, 조치 방법은 다음과 같습니다.

---

### 1. 핵심 의미: "파일 목록 노출"

보통 우리가 웹 사이트에 접속하면(`www.example.com`), 서버는 `index.html`이나 `default.aspx` 같은 **기본 문서(메인 페이지)**를 보여줍니다.

그런데 만약 사용자가 `www.example.com/images/` 처럼 **파일이 아닌 폴더 주소**까지 입력했는데, 해당 폴더에 메인 페이지가 없다면 어떻게 될까요?

* **설정 켜짐 (취약 - 리스팅 허용):**
* 브라우저 화면에 그 폴더 안에 있는 **모든 파일과 하위 폴더의 목록(이름, 날짜, 크기 등)이 텍스트 형태로 쫙 뜹니다.**
* 마치 윈도우 탐색기를 보는 것과 같습니다.


* **설정 꺼짐 (양호 - 리스팅 차단):**
* **"403 Forbidden (액세스가 거부되었습니다)"** 에러 메시지가 뜨며 아무것도 보여주지 않습니다. **(이게 정상입니다.)**



### 2. 점검 목적 (Why?)

"파일 목록 좀 보여주는 게 왜 위험한가요?"

해커에게 **서버의 내부 지도(Map)**를 쥐어주는 것과 같기 때문입니다.

1. **민감한 파일 노출:**
* 개발자가 실수로 남겨둔 `db_connect.bak` (백업 파일), `source.zip` (소스코드 압축 파일), `test.php` 등의 파일이 목록에 보이면, 해커는 바로 다운로드하여 비밀번호나 소스코드를 훔칩니다.


2. **구조 파악 용이:**
* 서버의 디렉터리 구조를 훤히 들여다보며 취약한 버전의 파일이 있는지, 어디를 공격하면 좋을지 쉽게 파악할 수 있습니다.



### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 판단 기준은 다음과 같습니다.

* **양호 (Safe):** IIS 설정에서 **"디렉터리 검색(Directory Browsing)"** 기능이 비활성화(Disabled)되어 있는 경우
* **취약 (Vulnerable):** 해당 기능이 활성화(Enabled)되어 있어, 브라우저에서 파일 목록이 보이는 경우

### 4. 설정 확인 및 조치 방법 (IIS 관리자)

이 설정은 IIS(웹 서버) 관리 콘솔에서 아주 쉽게 끌 수 있습니다.

1. **IIS 관리자 실행:** `Win + R` > `inetmgr` 입력
2. **사이트 선택:** 왼쪽 트리 메뉴에서 설정할 웹 사이트(또는 폴더)를 클릭합니다.
3. **기능 열기:**
* 중앙의 기능 보기 화면에서 **"디렉터리 검색 (Directory Browsing)"** 아이콘을 찾아 더블 클릭합니다.


4. **조치 (끄기):**
* 우측 작업(Actions) 패널에서 **[사용 안 함 (Disable)]**을 클릭합니다.
* *(이미 '사용' 버튼이 보인다면 현재 '사용 안 함' 상태인 것이므로 안전합니다.)*



### 요약

> **W-11 항목은 "웹 서버의 파일 목록을 남에게 보여주지 마라"는 뜻입니다.**

사용자는 개발자가 보여주기로 약속한 화면(웹 페이지)만 봐야지, 서버 안의 파일들을 자기 마음대로 뒤져보게 해서는 안 되기 때문입니다.

혹시 **"W-12 CGI 디렉터리 실행 권한 제거"**나 **"W-15 웹 서비스 파일 업로드 및 다운로드 제한"** 같은 다른 웹 서버 관련 항목도 궁금하신가요?
