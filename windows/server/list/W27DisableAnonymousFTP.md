KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드에 있는 **W-27 (W27DisableAnonymousFTP)** 항목은 **"익명(Anonymous) FTP 접속 금지"**를 의미합니다.

*(참고: 가이드 버전에 따라 W-25로 분류되기도 합니다.)*

한마디로 요약하면 **"아이디와 비밀번호가 없는 '신원 미상의 사용자'가 FTP 서버에 들어오지 못하게 막아라"**는 뜻입니다.

상세한 의미와 보안 위험성, 조치 방법은 다음과 같습니다.

---

### 1. 핵심 의미: "프리패스(Free Pass) 차단"

일반적인 FTP 접속은 `사용자 계정(ID)`과 `비밀번호(Password)`가 일치해야 로그인이 됩니다.
하지만 **익명(Anonymous) FTP** 기능이 켜져 있으면, 다음과 같이 아무나 접속할 수 있습니다.

* **아이디:** `anonymous` 또는 `ftp`
* **비밀번호:** `(없음)` 또는 `이메일 주소` (형식만 맞으면 아무거나 입력 가능)
* **결과:** 누구나 로그인 성공!

**W-27**은 이 기능을 **꺼서(Disable)**, 반드시 **허가된 계정**을 가진 사용자만 접속할 수 있게 하라는 것입니다.

### 2. 점검 목적 (Why?)

"공개 자료실처럼 쓰면 안 되나요?"

기업이나 기관의 내부 서버라면 절대 안 됩니다.

1. **무단 접근 및 정보 유출:**
* 익명 사용자가 접속하여 서버에 저장된 중요 문서나 소스 코드를 마음대로 열람하고 다운로드해 갈 수 있습니다.


2. **악성코드 유포지로 악용:**
* 만약 익명 사용자에게 **'쓰기(Write)'** 권한까지 있다면(W-26 취약), 해커는 악성코드나 불법 자료를 서버에 올리고, 다른 사람들에게 유포하는 **'파일 서버(Warez)'**로 악용합니다.


3. **공격의 발판:**
* 서버의 디렉터리 구조를 파악하거나, DoS 공격(디스크 꽉 채우기)을 수행하는 시발점이 됩니다.



### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 판단 기준은 다음과 같습니다.

* **양호 (Safe):** FTP 서비스의 **익명 접속(Anonymous Logon)** 기능이 **차단(Disabled)**된 경우
* **취약 (Vulnerable):** 익명 접속 기능이 **허용(Enabled)**되어 있는 경우

### 4. 설정 확인 및 조치 방법 (IIS 관리자)

윈도우 서버(IIS FTP)에서 이 설정을 끄는 방법은 다음과 같습니다.

1. **IIS 관리자 실행:** `inetmgr` 입력
2. **FTP 사이트 선택:** 설정할 FTP 사이트 클릭
3. **FTP 인증(FTP Authentication) 더블 클릭:**
* 중앙의 기능 보기 화면(IIS 섹션 아님, FTP 섹션)에 있습니다.


4. **설정 변경:**
* **익명 인증(Anonymous Authentication):** 상태가 **'사용(Enabled)'**이면 우클릭 후 **[사용 안 함(Disable)]**을 선택합니다.
* **기본 인증(Basic Authentication):** 상태가 **'사용(Enabled)'**이어야 실제 계정으로 로그인이 가능합니다.



### 💡 주의사항: W-26 (접근 권한)과의 관계

이 항목(W-27)은 **"문 단속"**이고, W-26은 **"방 안의 물건 단속"**입니다.

* **W-27 (익명 접속):** "신분증 없는 사람은 들어오지 마!" (문을 잠금)
* **W-26 (폴더 권한):** "들어온 사람이라도 아무거나 만지지 마!" (서랍을 잠금)

**가장 안전한 상태는 두 가지를 모두 조치하는 것입니다.** (익명 접속도 막고, 폴더 권한도 최소화함)

### 요약

> **W-27 항목은 "FTP 서버에 '투명 인간(익명 사용자)'은 출입 금지시켜라"는 뜻입니다.**

특별히 대국민 공개 자료실(Public Repository)을 운영하는 게 아니라면, 보안을 위해 무조건 꺼두어야 하는 기능입니다.

혹시 **FTP 접근 제어(IP 제한)**나 **FTP 서비스 배너 숨김** 같은 추가적인 FTP 보안 설정도 궁금하신가요?
