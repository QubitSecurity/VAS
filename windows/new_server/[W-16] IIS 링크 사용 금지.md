## [W-16] IIS 링크 사용 금지

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-16 |
| 항목명 | IIS 링크 사용 금지 |
| 위험도 | 상 (High) |
| 범주 | IIS 관리 |

### 점검 목적
심볼릭 링크(Symbolic Link)나 하드 링크가 웹 루트 내에 존재하는 경우, 공격자는 이를 통해 웹 루트 외부의 파일 시스템에 접근할 수 있습니다. 특히 심볼릭 링크를 통해 시스템 파일이나 설정 파일을 읽거나 덮어쓰는 공격이 가능하여 반드시 제한해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 웹 루트 내 심볼릭 링크 및 가상 디렉토리가 사용되지 않거나 엄격히 통제되는 경우 | 링크를 통한 웹 루트 외부 파일 접근 경로가 차단된다 |
| 취약 | 웹 루트 내에 외부 경로를 가리키는 심볼릭 링크가 존재하는 경우 | 링크를 통해 웹 루트 외부의 민감한 파일에 접근할 수 있다 |

### 참고 문서
- 심볼릭 링크(Symbolic link, 소프트 링크): 사용자가 심볼릭 링크 파일을 요청하면, 시스템이 해당 링크에 저장된 대상 경로를 따라가서 실제 원본 데이터를 가져와 전달 [WEB-12]
