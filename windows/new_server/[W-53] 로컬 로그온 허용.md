#### `W-53` `중` 로컬 로그온 허용

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-53 |
| 항목명 | 로컬 로그온 허용 |
| 위험도 | 중 (Medium) |
| 범주 | 패스워드 정책 |

### 점검 목적
로컬 로그온 권한이 불필요한 사용자나 그룹에 과도하게 부여된 경우, 물리적 접근이 가능한 공격자 또는 내부 위협자가 서버에 직접 로그인하여 시스템을 조작할 수 있습니다. 로컬 로그온은 반드시 필요한 관리자 계정에만 허용해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 로컬 로그온 권한이 Administrators 등 필요한 그룹에만 부여된 경우 | 불필요한 로컬 접근이 차단되어 내부 위협 및 물리적 접근 공격이 제한된다 |
| 취약 | 로컬 로그온 권한이 일반 사용자나 불필요한 그룹에 부여된 경우 | 서버에 물리적으로 접근한 공격자가 직접 로그인하여 시스템을 조작할 수 있다 |

### 참고 문서
- “로컬로 로그온 허용” 권한은 시스템 콘솔에 로그인을 허용하는 권한으로 반드시 콘솔 접근이 필요한 사용자 계정에만 해당 권한을 부여해야 함
- IIS 서비스를 사용할 경우 이 권한에 IUSR_<ComputerName > 계정을 할당함 [W-11]
