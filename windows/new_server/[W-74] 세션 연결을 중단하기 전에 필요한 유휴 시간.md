#### `W-74` `중` 세션 연결을 중단하기 전에 필요한 유휴시간

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-74 |
| 항목명 | 세션 연결을 중단하기 전에 필요한 유휴 시간 |
| 위험도 | 중 (Medium) |
| 범주 | 접근 제어 |

### 점검 목적
원격 세션의 유휴 타임아웃이 설정되지 않은 경우, 사용자가 세션을 방치해도 무기한 활성 상태로 유지되어 세션 하이재킹이나 방치된 세션 악용 위험이 있습니다. 적절한 유휴 타임아웃을 설정하여 방치된 세션을 자동으로 종료해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 세션 유휴 타임아웃이 30분 이하로 설정된 경우 | 방치된 세션이 자동으로 종료되어 무단 접근 위험이 감소한다 |
| 취약 | 세션 유휴 타임아웃이 설정되지 않은 경우 | 방치된 활성 세션을 통해 세션 하이재킹 공격이 가능하다 |

### 참고 문서
- Administrator는 해당 정책을 활성화하여 컴퓨터가 비활성 SMB Session을 중단하는 시점을 제어할 수 있으며, 클라이언트를 다시 시작하면 해당 Session은 자동으로 다시 연결됨. 해당 정책의 값을 0으로 설정하면 가능한 한 빨리 유휴 Session 연결은 끊어지며, 최댓값은 99999(208일)로 사실상 정책 설정 해제를 의미함
- SMB(서버 메시지 블록): LAN이나 컴퓨터 간의 통신에서 데이터 송수신을 하기 위한 프로토콜 [W-56]
