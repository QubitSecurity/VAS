#### `W-47` `중` 계정 잠금 기간 설정

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-47 |
| 항목명 | 계정 잠금 기간 설정 |
| 위험도 | 중 (Medium) |
| 범주 | 패스워드 정책 |

### 점검 목적
계정 잠금 임계값(W-04)이 설정되어 있더라도 잠금 기간이 너무 짧거나 0으로 설정된 경우, 공격자는 잠금이 자동 해제되는 시간을 노려 지속적으로 패스워드 대입 공격을 반복할 수 있습니다. 적절한 잠금 기간을 설정하여 공격 시도 자체를 비효율적으로 만들어야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 계정 잠금 기간이 30분 이상으로 설정된 경우 | 잠금 해제까지 충분한 시간이 부여되어 반복적인 대입 공격이 실질적으로 불가능해진다 |
| 취약 | 계정 잠금 기간이 설정되지 않았거나 너무 짧은 경우 | 짧은 간격으로 반복 대입 공격이 가능하여 계정 잠금 효과가 무력화된다 |

### 참고 문서 
- 계정 잠금 기간 설정은 계정 잠금 임계값을 초과한 사용자 계정이 잠기는 시간을 결정함. 잠긴 계정은 관리자가 재설정하거나 해당 계정의 잠금 유지 시간이 만료되어야 사용할 수 있음
- 계정 잠금 기간 설정을 사용하면 해당 기간 잠긴 계정은 사용할 수 없으며, 계정 잠금이 해제될 때까지 접근할 수 없음
- 계정 잠금 정책: 해당 계정이 시스템으로부터 잠기는 환경과 시간을 결정하는 정책으로 ‘계정 잠금 기간’, ‘계정 잠금 임계값’, ‘다음 시간 후 계정 잠금 수를 원래대로 설정’의 세 가지 하위 정책을 가짐 [W-08]
