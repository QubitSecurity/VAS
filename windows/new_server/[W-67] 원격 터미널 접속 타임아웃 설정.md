#### `W-67` `중` 원격터미널 접속 타임아웃 설정

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-67 |
| 항목명 | 원격 터미널 접속 타임아웃 설정 |
| 위험도 | 중 (Medium) |
| 범주 | 접근 제어 |

### 점검 목적
원격 터미널(RDP) 세션에 타임아웃이 설정되지 않은 경우, 사용자가 세션을 종료하지 않고 자리를 비워도 세션이 무기한 활성 상태로 유지됩니다. 방치된 활성 세션은 물리적 접근 공격자나 세션 하이재킹 공격자가 악용할 수 있으므로 적절한 유휴 세션 타임아웃을 설정해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 원격 터미널 유휴 세션 타임아웃이 30분 이하로 설정된 경우 | 일정 시간 미사용 시 세션이 자동 종료되어 방치된 세션을 통한 무단 접근이 차단된다 |
| 취약 | 원격 터미널 타임아웃이 설정되지 않은 경우 | 방치된 활성 세션을 통해 무단 접근 및 세션 하이재킹 공격이 가능하다 |

### 참고 문서
- 기반시설 시스템에서 원격터미널 서비스의 이용은 원칙적으로 금지하나, 부득이하게 해당 기능을 활용해야 하는 경우 접속 Timeout 설정 등의 보안 설정을 적용해야 함 [W-36]
