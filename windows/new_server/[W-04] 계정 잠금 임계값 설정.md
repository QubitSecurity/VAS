#### `W-04` `상` 계정 잠금 임계값 설정

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-04 |
| 항목명 | 계정 잠금 임계값 설정 |
| 위험도 | 상 (High) |
| 범주 | 계정 관리 |

### 점검 목적
계정 잠금 임계값이 설정되어 있지 않으면 공격자는 로그인 화면에서 수천~수만 번의 패스워드를 제한 없이 시도할 수 있습니다(무차별 대입 공격, Brute Force). 일정 횟수 이상 로그인 실패 시 계정을 잠그도록 설정하면 이러한 공격을 효과적으로 차단할 수 있습니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 계정 잠금 임계값이 5회 이하로 설정된 경우 | 일정 횟수 실패 시 계정이 잠겨 무차별 대입 공격을 효과적으로 차단한다 |
| 취약 | 계정 잠금 임계값이 6회 이상이거나 0(잠금 없음)으로 설정된 경우 | 패스워드를 무제한으로 시도할 수 있어 계정 탈취 위험이 크게 높아진다 |


### 참고 문서
- ※계정 잠금 임계값 설정은 사용자 계정이 잠기는 로그온 실패 횟수를 결정하며 잠긴 계정은 관리자가 재설정하거나 해당 계정의 잠금 유지 시간이 만료되어야 사용할 수 있음
※계정 잠금 정책: 해당 계정이 시스템으로부터 잠기는 환경과 시간을 결정하는 정책으로 ‘계정 잠금 기간’, ‘계정 잠금 임계값’, ‘다음 시간 후 계정 잠금 수를 원래대로 설정’의 세 가지 하위 정책을 가짐
※관련 점검 항목: W-08(중) [W-04]
