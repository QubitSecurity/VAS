## [W-11] IIS 디렉토리 리스팅 제거

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-11 |
| 항목명 | IIS 디렉토리 리스팅 제거 |
| 위험도 | 상 (High) |
| 범주 | IIS 관리 |

### 점검 목적
디렉토리 리스팅이 활성화된 경우, 웹 브라우저에서 특정 디렉토리 URL에 접근했을 때 해당 디렉토리의 파일 목록이 그대로 노출됩니다. 공격자는 이를 통해 설정 파일, 백업 파일, 소스코드 등 민감한 파일의 존재를 파악하고 직접 다운로드를 시도할 수 있습니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 디렉토리 검색(Directory Browsing)이 비활성화된 경우 | 파일 목록이 노출되지 않아 공격자가 내부 파일 구조를 파악할 수 없다 |
| 취약 | 디렉토리 검색(Directory Browsing)이 활성화된 경우 | 디렉토리 내 파일 목록이 외부에 노출되어 민감 파일 탈취 위험이 있다 |

### 참고 문서
- 보안을 위해 전역 설정에서 디렉터리 검색을 비활성화하고, 공유가 반드시 필요한 디렉터리에 대해서만 선별적으로 최소한의 접근 권한을 부여하여 운영해야 함.
- IIS 6.0 이하: 인터넷 정보 서비스 → 등록 정보 → 홈 디렉터리 → '디렉터리 검색' 체크 해제.
- IIS 7.0 이상: IIS 관리자 → 해당 웹 사이트 선택 → '디렉터리 검색' 기능 → '사용 안 함'으로 설정. [W-11]
