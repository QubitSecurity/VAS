[W-55] 최근 암호 기억

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-55 |
| 항목명 | 최근 암호 기억 |
| 위험도 | 중 (Medium) |
| 범주 | 패스워드 정책 |

### 점검 목적
패스워드 히스토리가 설정되지 않거나 기억 개수가 적은 경우, 사용자가 패스워드를 변경할 때 이전에 사용하던 패스워드를 그대로 재사용할 수 있어 패스워드 변경 정책이 무의미해집니다. 최소 12개 이상의 이전 패스워드를 기억하여 재사용을 방지해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 최근 패스워드 기억 개수가 12개 이상으로 설정된 경우 | 이전 패스워드 재사용이 충분히 제한되어 패스워드 변경 정책이 실효성을 가진다 |
| 취약 | 최근 패스워드 기억 개수가 설정되지 않았거나 12개 미만인 경우 | 적은 수의 패스워드만 기억하면 쉽게 이전 패스워드로 복귀할 수 있다 |
