## [W-13] IIS 상위 디렉토리 접근 금지

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-13 |
| 항목명 | IIS 상위 디렉토리 접근 금지 |
| 위험도 | 상 (High) |
| 범주 | IIS 관리 |

### 점검 목적
디렉토리 트래버설(Directory Traversal) 취약점은 `../../` 와 같은 경로 조작을 통해 웹 루트 밖의 파일 시스템에 접근하는 공격입니다. 상위 디렉토리 접근이 허용된 경우 공격자는 시스템 파일, 설정 파일 등 웹 루트 외부의 민감한 파일을 열람하거나 수정할 수 있습니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 상위 경로 허용(Enable Parent Paths)이 비활성화된 경우 | 디렉토리 트래버설 공격을 통한 웹 루트 외부 파일 접근이 차단된다 |
| 취약 | 상위 경로 허용이 활성화된 경우 | 경로 조작을 통해 시스템 내 임의 파일에 접근할 수 있다 |

### 참고 문서
- “..” 는 Unicode 버그, 서비스 거부와 같은 공격에 쉽게 이용되므로 허용하지 않는 것을 권장함 [WEB-06]
