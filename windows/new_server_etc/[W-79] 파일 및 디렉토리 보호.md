## [W-79] 파일 및 디렉토리 보호

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-79 |
| 항목명 | 파일 및 디렉토리 보호 |
| 위험도 | 중 (Medium) |
| 범주 | 파일 및 디렉토리 관리 |

### 점검 목적
시스템 파일, 설정 파일, 프로그램 파일 등 중요한 파일과 디렉토리에 적절한 ACL이 설정되지 않은 경우, 권한 상승 공격이나 악성코드에 의해 파일이 변조될 수 있습니다. 최소 권한 원칙에 따라 각 파일 및 디렉토리의 접근 권한을 엄격히 통제해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 시스템 파일 및 디렉토리에 최소 권한 ACL이 적용된 경우 | 권한 상승 공격이나 악성코드에 의한 파일 변조 위험이 감소한다 |
| 취약 | 시스템 파일 및 디렉토리에 Everyone, Users 등 과도한 권한이 부여된 경우 | 불필요한 계정에 의한 파일 변조 및 권한 상승 공격이 가능하다 |

### 참고 문서
- 기존에 FAT 파일 시스템을 사용하다가 NTFS로 변환하기 위해서는 convert.exe 명령을 사용할 수 
있지만, FAT 파일 시스템으로 운영 중 변환해야 하는 경우 Default ACL이 적용되지 않으므로 가능
한 초기 설치 시 NTFS 파일 시스템을 선택하는 것을 권장함
- 최근 운영체제 버전에서는 FAT32 파일 시스템을 지원하지 않으나 기존 FAT32에서 NTFS 변환 
가능
- NTFS, FAT 파일 시스템 비교: FAT32에는 NTFS가 제공하는 보안 기능이 없으므로 컴퓨터에 
FAT32 파티션 또는, 볼륨이 있는 경우 컴퓨터에 액세스 가능한 모든 사용자가 파일을 읽을 수 
있으며 FAT32에는 크기 제한이 있음. [W-61]
