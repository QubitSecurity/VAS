## [W-17] IIS 파일 업로드 및 다운로드 제한

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-17 |
| 항목명 | IIS 파일 업로드 및 다운로드 제한 |
| 위험도 | 상 (High) |
| 범주 | IIS 관리 |

### 점검 목적
파일 업로드 기능이 제한 없이 허용될 경우, 공격자가 웹쉘(WebShell) 등 악성 스크립트를 업로드하고 서버에서 실행하여 시스템을 장악할 수 있습니다. 업로드 가능한 파일 형식, 크기, 저장 경로를 엄격히 제한하고 업로드 디렉토리에서의 스크립트 실행을 차단해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 업로드 디렉토리에 실행 권한이 없고, 허용 파일 형식이 제한된 경우 | 악성 스크립트 업로드 및 실행을 통한 서버 장악이 차단된다 |
| 취약 | 업로드 디렉토리에 스크립트 실행 권한이 부여된 경우 | 업로드된 웹쉘 실행을 통해 서버 전체가 장악될 수 있다 |

### 참고 문서
- 용량 및 경로 제한이 미흡할 경우, 공격자가 웹쉘을 업로드하여 시스템 권한을 탈취하거나 대용량 파일 반복 업로드로 서버 장애를 유발할 수 있습니다. [WEB-08, WEB-24]
