## [W-43] Autologon 기능 제어

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-43 |
| 항목명 | Autologon 기능 제어 |
| 위험도 | 상 (High) |
| 범주 | 보안 설정 |

### 점검 목적
Autologon 기능이 활성화된 경우 레지스트리에 계정명과 패스워드가 평문 또는 약한 인코딩으로 저장됩니다. 공격자가 레지스트리에 접근하거나 시스템을 재부팅하면 저장된 자격증명을 즉시 탈취하거나 자동으로 관리자 세션에 접근할 수 있습니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | Autologon 기능이 비활성화된 경우 | 자동 로그온을 통한 무단 세션 접근 및 자격증명 노출 위험이 차단된다 |
| 취약 | Autologon 기능이 활성화되어 레지스트리에 자격증명이 저장된 경우 | 레지스트리 접근 또는 재부팅을 통해 자격증명이 즉시 탈취될 수 있다 |

### 참고 문서
- Autologon: 레지스트리에 암호화되어 저장된 대체 증명을 사용하여 자동으로 로그인하는 기능
- AutoAdminLogon 키가 없는 경우 기본값 비활성화 [W-52]
