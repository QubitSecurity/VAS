## [W-22] IIS Exec 명령어 쉘 호출 진단

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-22 |
| 항목명 | IIS Exec 명령어 쉘 호출 진단 |
| 위험도 | 상 (High) |
| 범주 | IIS 관리 |

### 점검 목적
SSI(Server Side Include)의 `#exec` 명령은 웹 페이지에서 서버의 시스템 명령어를 직접 실행할 수 있는 기능입니다. 이 기능이 허용된 경우 공격자가 SSI 인젝션을 통해 서버에서 임의의 명령을 실행하는 원격 코드 실행(RCE) 공격이 가능합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | SSI Exec 기능이 비활성화된 경우 | SSI를 통한 시스템 명령 실행 경로가 차단된다 |
| 취약 | SSI Exec 기능이 활성화된 경우 | SSI 인젝션을 통해 서버에서 임의 명령 실행이 가능하다 |

### 참고 문서
- SSI(Server-Side Includes): CGI 프로그램을 작성하거나 혹은 서버 사이드 스크립트를 사용하는 언어로, 웹 서버가 사용자에게 페이지를 제공하기 전에 구문을 해석하도록 지시하는 역할을 함
※SSI(Server-Side Includes) Injection: HTML 문서 내 입력받은 변수값을 서버 측에서 처리할 때 부적절한 명령문이 포함 및 실행되어 서버의 데이터가 유출되는 취약점 [WEB-19]
