## [W-21] IIS 미사용 스크립트 매핑 제거

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-21 |
| 항목명 | IIS 미사용 스크립트 매핑 제거 |
| 위험도 | 상 (High) |
| 범주 | IIS 관리 |

### 점검 목적
IIS는 .asp, .aspx, .php 등 다양한 스크립트 확장자에 대한 처리기 매핑을 기본으로 등록합니다. 실제로 사용하지 않는 스크립트 매핑이 활성화된 경우, 해당 확장자를 처리하는 엔진의 취약점이 악용될 수 있습니다. 사용하지 않는 스크립트 매핑은 모두 제거해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 실제 사용하는 스크립트 매핑만 활성화된 경우 | 불필요한 스크립트 처리 엔진의 취약점 노출이 차단된다 |
| 취약 | 사용하지 않는 스크립트 매핑이 활성화된 경우 | 미사용 스크립트 처리 엔진의 취약점이 공격자에게 악용될 수 있다 |

### 참고 문서
-  .asp나 .shtm과 같은 확장자들은 특정 DLL 파일과 매핑되어 있어, 이러한 파일들에 대한 요청이 들어오면 해당 DLL에 의해 처리됨
- 스크립트 매핑: 웹 서버에서 특정 파일 확장자나 URL 경로를 특정 스크립트 처리프로그램에 연결하는 설정 [WEB-15]
