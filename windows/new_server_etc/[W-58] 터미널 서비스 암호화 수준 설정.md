## [W-58] 터미널 서비스 암호화 수준 설정

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-58 |
| 항목명 | 터미널 서비스 암호화 수준 설정 |
| 위험도 | 중 (Medium) |
| 범주 | 접근 제어 |

### 점검 목적
RDP 세션의 암호화 수준이 낮게 설정된 경우, 네트워크 스니핑을 통해 세션 데이터, 자격증명, 입력 정보 등이 노출될 수 있습니다. RDP 암호화 수준은 반드시 높음(High) 이상으로 설정하거나 TLS/SSL을 사용하여 세션 데이터를 보호해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 터미널 서비스 암호화 수준이 높음(High) 또는 FIPS 규격으로 설정된 경우 | 세션 데이터가 강력한 암호화로 보호되어 네트워크 스니핑을 통한 정보 노출이 차단된다 |
| 취약 | 암호화 수준이 낮음(Low) 또는 클라이언트 호환(Client Compatible)으로 설정된 경우 | 약한 암호화로 인해 세션 데이터가 네트워크 스니핑에 노출될 수 있다 |

### 참고 문서
- 기반시설 시스템은 원격 데스크톱 서비스의 사용을 원칙적으로 금지하나, 부득이 해당 서비스를 사
용해야 하는 경우 클라이언트 서버 간의 데이터 전송 시 암호화하여 보호해야 함 [W-28]
