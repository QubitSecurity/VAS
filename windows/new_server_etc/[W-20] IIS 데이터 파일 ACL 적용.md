## [W-20] IIS 데이터 파일 ACL 적용

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-20 |
| 항목명 | IIS 데이터 파일 ACL 적용 |
| 위험도 | 상 (High) |
| 범주 | IIS 관리 |

### 점검 목적
웹 서버에서 사용하는 데이터 파일(설정 파일, 로그, DB 파일 등)에 적절한 ACL(Access Control List)이 설정되지 않은 경우, 웹 프로세스 탈취 시 공격자가 해당 파일에 접근하거나 수정할 수 있습니다. 파일별로 웹 프로세스 계정에 필요한 최소 권한만 부여해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 웹 데이터 파일에 최소 권한 ACL이 적용된 경우 | 웹 프로세스 탈취 시에도 접근 가능한 파일 범위가 제한된다 |
| 취약 | 웹 데이터 파일에 Everyone 또는 과도한 권한이 부여된 경우 | 웹 프로세스 탈취 시 모든 파일에 무제한 접근이 가능하다 |

### 참고 문서
- 웹 서비스 경로 내의 파일에 비인가자가 접근 가능할 경우, 해당 파일의 수정 및 삭제로 인해 웹 서비스 운영 장애가 발생하거나 계정 비밀번호 정보 등 중요한 데이터가 유출될 위험이 존재합니다. 근 권한은 업무 수행에 반드시 필요한 때에만 허용하고, 필요한 특정 위치의 특정 사용자에게만 최소 권한을 부여하도록 조치해야 합니다.[WEB-14, 26]
