KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드에 있는 **W-14 (W14RemoveUnnecessaryIISFiles)** 항목은 **"IIS 불필요 파일 제거"**를 의미합니다.

한마디로 요약하면 **"IIS를 처음 설치할 때 딸려온 '샘플 사이트', '설명서(도움말)', '예제 스크립트' 등을 싹 다 지워라"**는 뜻입니다.

상세한 의미와 왜 지워야 하는지, 어떤 파일이 대상인지 설명해 드리겠습니다.

---

### 1. 핵심 의미: "청소하지 않은 새 집"

윈도우 서버에 IIS(웹 서버)를 설치하면, 마이크로소프트는 사용자가 기능을 쉽게 익힐 수 있도록 친절하게 각종 **샘플 파일**과 **설명서**를 같이 설치해줍니다. (특히 구버전 IIS에서 심함)

* **샘플 파일:** "홈페이지는 이렇게 만드세요"라고 보여주는 예제 코드 (`.asp`, `.aspx` 등)
* **도움말/매뉴얼:** IIS 사용 설명서 웹 페이지
* **관리 스크립트:** 서버를 관리하기 위한 도구들 (`/adminscripts`)

**W-14**는 실제 서비스 운영에는 1%도 필요 없는 이 파일들이 **서버에 남아있는지** 점검하는 항목입니다.

### 2. 점검 목적 (Why?)

"안 쓰는 파일 좀 놔두면 어때서요?"

이 파일들은 해커들에게 **보물창고**나 다름없기 때문입니다.

1. **취약한 코드 (XSS 등):** 샘플 파일은 기능을 보여주기 위해 대충 만들어진 경우가 많아, 보안 코딩이 안 되어 있습니다. 해커는 이 샘플 파일의 취약점을 이용해 스크립트를 실행하거나 서버 정보를 빼냅니다.
2. **서버 정보 노출:** 도움말이나 기본 페이지를 통해 "아, 이 서버는 IIS 7.5 버전을 쓰는구나"라고 버전을 특정할 수 있습니다.
3. **경로 파악:** `C:\Inetpub\Scripts` 같은 기본 경로가 존재한다는 확신을 주어 공격의 발판이 됩니다.

### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따르면 다음의 디렉터리(폴더)가 존재하면 취약한 것으로 봅니다.

* **양호 (Safe):** 아래의 불필요한 디렉터리가 모두 삭제된 경우
* **취약 (Vulnerable):** 기본 샘플, 설명서 등의 디렉터리가 하나라도 남아있는 경우

### 4. 삭제 대상 (주요 폴더 목록)

보통 `C:\Inetpub\` 폴더 아래에 위치합니다. 실제 사용하는 웹 소스가 아니라면 다음 폴더들은 과감히 삭제해야 합니다.

* **Samples:** `\IISSamples` (샘플 사이트)
* **Help:** `\IISHelp` (도움말)
* **Scripts:** `\Scripts` (예제 스크립트)
* **AdminScripts:** `\AdminScripts` (관리용 스크립트 - 매우 위험)
* **_vti_bin:** (FrontPage Server Extensions 관련 - 사용 안 하면 삭제)
* **Printers:** (웹 프린팅 기능 - 사용 안 하면 삭제)

### 5. 설정 확인 및 조치 방법

가장 확실한 방법은 파일 탐색기에서 해당 폴더를 찾아 **Shift + Delete(영구 삭제)** 하는 것입니다.

1. **파일 탐색기 열기:** `Win + E`
2. **경로 이동:** `C:\Inetpub\` (또는 웹 루트 경로)
3. **폴더 확인:** 위에 언급된 `IISSamples`, `AdminScripts` 등의 폴더가 있는지 확인합니다.
4. **조치:** 해당 폴더를 삭제합니다.

### 💡 참고: 최신 IIS (Windows Server 2012 이후)

다행히 최신 버전의 윈도우 서버(IIS 8.0 이상)부터는 보안이 강화되어, 설치 시 이러한 샘플 파일들이 **기본적으로 설치되지 않는 경우가 많습니다.**

하지만 **업그레이드 설치**를 했거나, 관리자가 실수로 **"IIS 예제" 기능을 체크해서 설치**했을 수도 있으므로 반드시 눈으로 확인하고 지우는 절차가 필요합니다.

### 요약
