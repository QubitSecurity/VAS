## [W-07]() 공유 권한 및 사용자 그룹 설정 (네트워크_스크립트)

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-07 |
| 항목명 | 공유 권한 및 사용자 그룹 설정 |
| 위험도 | 상 (High) |
| 범주 | 공유 및 권한 |

### 점검 목적
네트워크 공유 폴더에 Everyone 그룹이 포함되거나 쓰기 권한이 과도하게 부여된 경우, 인증된 내부 사용자뿐 아니라 인증 우회에 성공한 공격자도 파일을 읽거나 악성코드를 업로드할 수 있습니다. 공유 폴더는 반드시 필요한 사용자와 그룹에만 최소 권한으로 설정해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 공유 폴더에 Everyone 그룹이 없고, 필요한 사용자/그룹에만 최소 권한이 부여된 경우 | 불필요한 접근이 차단되어 정보 유출 및 악성코드 업로드 위험이 감소한다 |
| 취약 | 공유 폴더에 Everyone 그룹이 포함되거나 쓰기 권한이 불필요하게 부여된 경우 | 모든 사용자가 공유 자원에 접근 가능하여 정보 유출 및 시스템 침해 위험이 높아진다 |



### 참고 문서
- "응용프로그램이나 백업 용도로 반드시 공유가 필요한 경우에는 Everyone 권한 대신 특정 관리자 계정이나 서비스 전용 계정을 생성하여 최소 권한으로 할당할 것을 권고함." [W-16]

### 개요

| 위험도 | 상 (High) |

### 점검 목적
IIS(Internet Information Services)는 웹 서비스를 제공하는 서버 소프트웨어로, 불필요하게 구동 중인 경우 외부 공격의 주요 진입점이 됩니다. 웹 서비스가 필요하지 않은 서버에서 IIS가 실행 중이라면 즉시 중지해야 하며, 필요한 경우라도 보안 설정을 강화해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | IIS 서비스가 비활성화되어 있거나, 업무상 필요에 의해 운영되며 보안 설정이 적용된 경우 | 불필요한 웹 서비스 노출이 없어 외부 공격 표면이 감소한다 |
| 취약 | 업무상 불필요한 IIS 서비스가 실행 중인 경우 | 불필요한 웹 서버가 외부 공격의 진입점으로 악용될 수 있다 |


### 참고 문서
- 특별한 목적으로 웹 서비스를 사용하는 경우가 아니라면 서비스를 중지하고 '시작 유형'을 '사용 안 함'으로 설정하여 잠재적인 취약점 노출을 차단해야 한다.
- 제어판의 서비스 관리 도구(services.msc)를 통해 해당 서비스의 설치 및 구동 여부를 확인할 수 있다. [W-19]

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-24 |
| 항목명 | NetBIOS 바인딩 서비스 구동 점검 |
| 위험도 | 상 (High) |
| 범주 | 네트워크 서비스 |

### 점검 목적
NetBIOS는 구형 Windows 네트워크 환경에서 사용되던 프로토콜로, 현재 인터넷 환경에서는 불필요한 경우가 대부분입니다. NetBIOS가 활성화된 경우 공격자는 NetBIOS 이름 조회를 통해 내부 네트워크 구조, 계정 정보, 공유 자원을 수집할 수 있으며 NBNS 스푸핑 공격의 대상이 될 수 있습니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | NetBIOS over TCP/IP가 비활성화된 경우 | NetBIOS를 통한 정보 수집 및 스푸핑 공격 경로가 차단된다 |
| 취약 | NetBIOS over TCP/IP가 활성화된 경우 | 내부 네트워크 정보 노출 및 스푸핑 공격에 악용될 수 있다 |

### 참고 문서
- NetBIOS(Network Basic Input/Output System)는 별개의 컴퓨터상에 있는 응용프로그램들이 근거리통신망 내에서 서로 통신할 수 있게 해주는 API. 이름 서비스, 세션 서비스, 데이터그램 서비스를 제공하며, 주로 네트워크 장치 간의 통신을 지원 [W-20]

## [W-31] 최신 서비스팩 적용 (보안_스크립트)

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-31 |
| 항목명 | 최신 서비스팩 적용 |
| 위험도 | 상 (High) |
| 범주 | 패치 관리 |

### 점검 목적
서비스팩(Service Pack)은 OS에 누적된 보안 패치, 버그 수정, 성능 개선을 통합한 대규모 업데이트입니다. 최신 서비스팩이 적용되지 않은 경우 공개된 취약점이 패치되지 않은 상태로 남아 공격자가 해당 취약점을 직접 악용할 수 있습니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | OS에 최신 서비스팩이 적용된 경우 | 공개된 주요 취약점이 패치되어 알려진 공격에 대한 방어가 이루어진다 |
| 취약 | 최신 서비스팩이 적용되지 않은 경우 | 패치되지 않은 취약점을 통한 공격에 노출될 수 있다 |

### 참고 문서
- 서비스 팩: 운영체제 응용프로그램의 기능 추가 및 버그나 보안 취약점을 해결한 패치 파일을 단일 묶음으로 배포하는 패키지
- 익스플로잇: 컴퓨터의 소프트웨어나 하드웨어 또는 컴퓨터 관련 전자 제품의 버그, 보안 취약점 등 설계상 결함을 이용해 공격자의 의도된 동작을 수행하도록 만들어진 프로그램 [PC-11]

## [W-32] 최신 HOT FIX 적용 (보안 스크립트)

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-32 |
| 항목명 | 최신 HOT FIX 적용 |
| 위험도 | 상 (High) |
| 범주 | 패치 관리 |

### 점검 목적
핫픽스(Hot Fix)는 서비스팩 사이에 개별적으로 배포되는 긴급 보안 패치입니다. 특히 원격 코드 실행(RCE), 권한 상승 등 위험도가 높은 취약점에 대한 긴급 패치가 포함되므로 신속하게 적용해야 합니다. 미적용 상태에서는 패치 공개 이후 해당 취약점을 노린 공격이 급증합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 최신 보안 핫픽스가 모두 적용된 경우 | 알려진 개별 취약점에 대한 방어가 최신 상태로 유지된다 |
| 취약 | 보안 관련 핫픽스가 적용되지 않은 항목이 존재하는 경우 | 공개된 취약점을 통한 원격 공격에 노출될 수 있다 |

### 참고 문서
- 보안 패치보다 취약성을 이용한 공격 도구가 먼저 출현할 수 있으므로 보안 패치는 발표 후 가능한 한 빨리 설치할 것을 권장함 [W-38]

| 취약 | 백신 정의 파일이 오래되었거나 업데이트가 비활성화된 경우 | 신규 악성코드 탐지가 불가능하여 감염 위험이 높아진다 |

| 범주 | 보안 설정 |

### 점검 목적
보안 감사 로그를 기록할 수 없을 때 시스템을 강제 종료하는 정책이 활성화된 경우, 공격자가 의도적으로 로그 공간을 고갈시켜 서비스 중단(DoS)을 유발할 수 있습니다. 가용성이 중요한 서버 환경에서는 이 정책을 비활성화하되, 별도의 로그 관리 방안을 마련해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 해당 정책이 비활성화된 경우 | 로그 공간 고갈로 인한 의도적 서비스 중단 공격을 방지할 수 있다 |
| 취약 | 해당 정책이 활성화된 경우 | 로그 저장 공간 고갈 시 시스템이 자동 종료되어 서비스 중단이 유발될 수 있다 |

### 참고 문서

복구하려면 관리자가 로그온하여 로그를 보관한 다음 로그를 지우고 이 옵션을 원하는 대로 다시 설정해야 합니다. 이 보안 설정을 다시 설정할 때까지는 보안 로그가 꽉 차지 않았더라도 Administrators 그룹의 구성원이 아니면 어떤 사용자도 시스템에 로그온할 수 없습니다. [W-50]

### 점검 목적
Telnet은 모든 데이터를 평문으로 전송하는 구형 원격 접속 프로토콜입니다. Telnet을 사용하는 경우 네트워크 스니핑을 통해 자격증명과 세션 데이터 전체가 노출됩니다. Telnet 서비스는 비활성화하고 SSH 또는 RDP(TLS 적용)로 대체해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | Telnet 서비스가 비활성화된 경우 | 평문 전송을 통한 자격증명 및 세션 데이터 노출 위험이 차단된다 |
| 취약 | Telnet 서비스가 실행 중인 경우 | 자격증명 및 세션 데이터 전체가 네트워크 스니핑에 노출될 수 있다 |

### 참고 문서
- Windows 서버의 Telnet 서비스의 두 가지 인증 방법
Ÿ NTLM 인증: 암호를 전송하지 않고 negotiate/challenge/response 절차로 인증 수행
Ÿ Password 인증: 관리자 및 Telnet Clients 그룹에 포함된 ID/PW로 인증 수행
- Windows 2016 이상 버전에서는 보안상 이슈로 인해 Telnet 서버 설치 제공하지 않음 [W-34]

## [W-67] 원격 터미널 접속 타임아웃 설정 (네트워크_스크립트)

| 항목 | 내용 |
|------|------|
| 점검 코드 | W-67 |
| 항목명 | 원격 터미널 접속 타임아웃 설정 |
| 위험도 | 중 (Medium) |
| 범주 | 접근 제어 |

### 점검 목적
원격 터미널(RDP) 세션에 타임아웃이 설정되지 않은 경우, 사용자가 세션을 종료하지 않고 자리를 비워도 세션이 무기한 활성 상태로 유지됩니다. 방치된 활성 세션은 물리적 접근 공격자나 세션 하이재킹 공격자가 악용할 수 있으므로 적절한 유휴 세션 타임아웃을 설정해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 원격 터미널 유휴 세션 타임아웃이 30분 이하로 설정된 경우 | 일정 시간 미사용 시 세션이 자동 종료되어 방치된 세션을 통한 무단 접근이 차단된다 |
| 취약 | 원격 터미널 타임아웃이 설정되지 않은 경우 | 방치된 활성 세션을 통해 무단 접근 및 세션 하이재킹 공격이 가능하다 |

### 참고 문서
- 기반시설 시스템에서 원격터미널 서비스의 이용은 원칙적으로 금지하나, 부득이하게 해당 기능을 활용해야 하는 경우 접속 Timeout 설정 등의 보안 설정을 적용해야 함 [W-36]

## [W-68] 예약된 작업에 의심스러운 명령이 등록되어 있는지 점검 (실행_스크립트)

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-68 |
| 항목명 | 예약된 작업에 의심스러운 명령이 등록되어 있는지 점검 |
| 위험도 | 중 (Medium) |
| 범주 | 로그 및 감사 |

### 점검 목적
Task Scheduler를 통해 등록된 작업은 시스템 재부팅 시 자동 실행되므로, 공격자가 악성 스크립트나 명령을 예약 작업으로 등록하면 지속적인 권한 상승 또는 데이터 유출을 수행할 수 있습니다. 정기적으로 예약 작업을 검토하여 의심스러운 작업이 등록되어 있는지 확인해야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 모든 예약 작업이 업무상 정당한 작업만 등록되어 있는 경우 | 악성 명령을 통한 지속적 권한 상승이나 데이터 유출 경로가 없다 |
| 취약 | 예약 작업에 의심스러운 명령어, 외부 URL 호출, PowerShell 다운로드 등이 포함된 경우 | 재부팅 시 자동 실행되는 악성 명령으로 시스템이 장악될 수 있다 |

## [W-69] 정책에 따른 시스템 로깅 설정 (로그_스크립트)

### 개요
| 항목 | 내용 |
|------|------|
| 점검 코드 | W-69 |
| 항목명 | 정책에 따른 시스템 로깅 설정 |
| 위험도 | 중 (Medium) |
| 범주 | 로그 및 감사 |

### 점검 목적
보안 감사 정책이 설정되지 않은 경우, 로그인 시도, 권한 변경, 파일 접근 등의 중요한 보안 이벤트가 기록되지 않아 침해 사고 발생 시 사후 분석이 불가능합니다. 정책에 따라 중요한 보안 이벤트를 감사하도록 설정하여 이상 징후를 추적할 수 있어야 합니다.

### 점검 기준
| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 중요한 보안 이벤트(로그온, 권한 변경, 파일 접근 등)에 대한 감사 정책이 활성화된 경우 | 보안 이벤트가 기록되어 침해 사고 시 사후 분석이 가능하다 |
| 취약 | 보안 감사 정책이 비활성화된 경우 | 중요한 보안 이벤트가 기록되지 않아 침해 추적이 불가능하다 |

### 참고 문서
- 감사 정책을 과도하게 설정할 경우, 보안 로그에 불필요한 항목이 많이 기록되므로 중요한 감사 항목 식별이 어려울 수 있으며, 시스템 성능에도 심각한 영향을 줄 수 있으므로 법적 요구 사항과 조직의 정책에 따라 꼭 필요한 로그를 남기도록 설정해야 함
- Windows 시스템은 보안 로그가 가득 차게 되는 경우 가장 오래된 감사 항목이 덮어 씌워짐 [W-40]
