KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드에 있는 **W-61 (W61EnforceSNMPCommunityStringComplexity)** 항목은 **"SNMP 커뮤니티 스트링(Community String)의 복잡성 설정"**을 의미합니다.

한마디로 요약하면 **"SNMP 서비스에서 사용하는 비밀번호(Community String)를 누구나 다 아는 'public'이나 'private'으로 쓰지 말고, 남들이 추측하기 어려운 복잡한 암호로 바꿔라"**는 뜻입니다.

상세한 의미와 왜 이것이 중요한지, 어떻게 설정해야 하는지 설명해 드리겠습니다.

---

### 1. 핵심 의미: "기본 비밀번호 금지"

SNMP(Simple Network Management Protocol)에서 **'커뮤니티 스트링(Community String)'**은 관리 서버(NMS)와 대상 서버가 통신할 때 사용하는 **"암호(Password)"** 역할을 합니다.

문제는 SNMP를 설치하면 이 암호가 전 세계 공통 기본값으로 설정된다는 점입니다.

* **취약한 설정 (기본값):**
* **`public`**: 읽기 전용 (Read Only) 권한의 기본 암호.
* **`private`**: 읽기/쓰기 (Read Write) 권한의 기본 암호.
* 해커는 서버를 발견하면 무조건 이 두 단어부터 넣어봅니다.


* **안전한 설정 (복잡하게 변경):**
* `P@ssw0rd_SNMP_2024!` 처럼 영문, 숫자, 특수문자를 섞어서 유추할 수 없게 변경한 상태입니다.



### 2. 점검 목적 (Why?)

"비밀번호가 쉬우면 어떤 일이 생기나요?"

1. **정보 유출 (Information Disclosure):**
* `public` 스트링이 노출되면, 해커는 서버의 시스템 정보, 네트워크 연결 상태, 실행 중인 프로세스 목록, 사용자 계정 정보 등을 모두 열람할 수 있습니다. (정찰 공격)


2. **시스템 변조 및 제어권 탈취:**
* `private` (쓰기 권한) 스트링이 노출되면, 해커가 원격에서 서버의 설정을 바꾸거나(라우팅 테이블 변경 등), 장비를 재부팅시키거나, 심지어 특정 서비스를 중지시킬 수도 있습니다.



### 3. 점검 기준 (Pass/Fail)

KISA 가이드라인에 따른 판단 기준은 다음과 같습니다.

* **양호 (Safe):**
* SNMP 서비스를 **사용하지 않는 경우** (가장 좋음)
* 사용하더라도 Community String이 **`public`, `private**` 등의 기본값이 아니며, **유추하기 어려운 문자열**로 설정된 경우


* **취약 (Vulnerable):**
* SNMP 서비스가 실행 중이고, Community String이 **`public`** 또는 **`private`**으로 설정된 경우



### 4. 설정 확인 및 조치 방법 (서비스 설정)

SNMP를 사용해야 한다면, 반드시 기본값을 지우고 새 값을 넣어야 합니다.

1. **서비스 속성 열기:**
* `Win + R` > `services.msc` 실행
* **SNMP Service**를 찾아 더블 클릭


2. **보안 탭 이동:**
* 상단의 **[보안(Security)]** 탭을 클릭합니다.


3. **커뮤니티 이름 확인 및 변경:**
* 목록에 **`public`**이나 **`private`**이 있다면 선택하고 **[제거(Remove)]**를 클릭합니다.
* **[추가(Add)]**를 클릭하여 새로운 커뮤니티 이름을 입력합니다.
* **권한:** 보통 모니터링만 한다면 **'읽기 전용(READ ONLY)'**을 선택합니다.
* **이름:** `SysMon_#99_Secret` 같이 복잡하게 만듭니다.





### 💡 권장 사항: SNMPv3 사용

이 항목에서 다루는 '커뮤니티 스트링' 방식은 **SNMP v1, v2c**라는 구형 버전의 이야기입니다. 이 버전들은 암호가 네트워크상에서 평문(Clear Text)으로 전송되므로, 스니핑(도청)에 취약합니다.

* 가능하다면 보안성이 훨씬 강력한 **SNMP v3 (사용자 인증 + 암호화 전송)**를 사용하는 것이 근본적인 해결책입니다.

### 요약

> **W-61 항목은 "SNMP 비밀번호를 '1234'처럼 뻔한 것(public)으로 쓰지 마라"는 뜻입니다.**

이것은 W-60(SNMP 실행 여부)과 함께 묶여서, **"안 쓰면 끄고(W-60), 쓸 거면 비밀번호 어렵게 바꿔라(W-61)"**는 세트 항목으로 이해하시면 됩니다.
