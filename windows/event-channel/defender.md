윈도우 디펜더(Microsoft Defender Antivirus)의 이벤트 로그는 보안 모니터링 및 트러블슈팅에 매우 중요한 역할을 합니다. 요청하신 것처럼 설정 변경, 탐지, 검사, 업데이트 등 목적에 따라 주요 이벤트 ID(Event ID)를 범주별로 정리해 드립니다.

디펜더의 주요 이벤트 로그는 **[이벤트 뷰어] -> [응용 프로그램 및 서비스 로그] -> [Microsoft] -> [Windows] -> [Windows Defender] -> [Operational]** 경로에서 확인할 수 있습니다.

---

### 1. 보호 상태 및 설정 변경 (Configuration & Status)

질문에서 언급하신 설정 변경 및 변조 방지 관련 이벤트가 포함된 가장 중요한 섹션입니다.

| 이벤트 ID | 설명 및 의미 |
| --- | --- |
| **5000** | **디펜더 활성화됨**: Microsoft Defender 바이러스 백신이 켜졌음을 나타냅니다. |
| **5001** | **디펜더 비활성화됨**: 다른 백신이 설치되었거나 정책에 의해 디펜더가 꺼졌을 때 기록됩니다. (설치 직후 OFF 확인 시 유용) |
| **5004** | **실시간 보호 상태 변경**: 실시간 보호 기능이 켜지거나 꺼졌을 때 기록됩니다. |
| **5007** | **설정 변경**: 레지스트리나 정책 등을 통해 디펜더 설정이 변경되었을 때 기록됩니다. (어떤 키/값이 변경되었는지 세부 정보에 표시됨) |
| **5013** | **변조 방지(Tamper Protection) 차단**: 누군가(또는 악성코드가) 임의로 디펜더의 핵심 설정을 끄거나 변경하려다 변조 방지 기능에 의해 막혔을 때 기록됩니다. |
| **1150** | **실시간 보호 설정 변경**: 예외 처리(Exclusion) 추가 등 실시간 보호와 관련된 세부 설정이 변경되었을 때 나타납니다. |
| **1151** | **실시간 보호 켜짐/꺼짐**: 사용자나 관리자에 의해 실시간 보호가 명시적으로 켜지거나 꺼진 상태를 나타냅니다. |

### 2. 위협 탐지 및 조치 (Malware Detection & Remediation)

악성코드가 발견되었거나 디펜더가 이를 처리한 기록입니다.

| 이벤트 ID | 설명 및 의미 |
| --- | --- |
| **1006 / 1116** | **악성코드 탐지**: 디펜더가 악성코드나 원치 않는 소프트웨어(PUA)를 발견했습니다. (1116은 주로 활성 위협 경고에 사용됨) |
| **1007 / 1117** | **악성코드 조치 완료**: 탐지된 위협에 대해 격리, 삭제 등의 조치를 성공적으로 수행했습니다. |
| **1015** | **의심스러운 동작 탐지**: 악성코드의 시그니처가 없더라도, 동작 모니터링(Behavior monitoring)을 통해 의심스러운 프로세스 행위를 차단했을 때 기록됩니다. |
| **1118** | **악성코드 조치 실패**: 위협을 발견했으나 파일 잠금, 권한 부족 등의 이유로 제거 또는 격리에 실패했습니다. |
| **1119** | **재부팅 필요**: 악성코드를 성공적으로 조치했으나, 완벽한 제거를 위해 시스템 재부팅이 필요합니다. |

### 3. 검사 상태 (Scan Operations)

수동 검사, 예약 검사 등의 시작 및 종료 상태를 나타냅니다.

| 이벤트 ID | 설명 및 의미 |
| --- | --- |
| **1000** | **검사 시작**: 빠른 검사, 전체 검사 또는 사용자 지정 검사가 시작되었습니다. |
| **1001** | **검사 완료**: 검사가 정상적으로 완료되었습니다. |
| **1002** | **검사 취소/중지**: 검사 도중 사용자나 시스템에 의해 취소되었습니다. |
| **1005** | **검사 실패**: 오류로 인해 검사를 완료하지 못했습니다. |

### 4. 보안 인텔리전스(시그니처) 업데이트 (Updates)

백신 엔진 및 바이러스 정의 업데이트 관련 로그입니다.

| 이벤트 ID | 설명 및 의미 |
| --- | --- |
| **2000** | **업데이트 성공**: 보안 인텔리전스(시그니처)가 성공적으로 업데이트되었습니다. |
| **2001** | **업데이트 실패**: 네트워크 문제 등으로 인해 시그니처 업데이트를 실패했습니다. |
| **2002** | **엔진 업데이트 완료**: 악성코드 탐지 엔진 자체가 성공적으로 업데이트되었습니다. |
| **2010 / 2011** | 각각 업데이트를 확인하기 시작했음(2010), 확인할 업데이트가 없음(2011)을 의미합니다. |

---

> **💡 참고 팁:**
> 엔터프라이즈 환경이나 여러 대의 PC를 관리하실 때는 일일이 이벤트 뷰어를 여는 것보다 **PowerShell**을 사용하는 것이 훨씬 빠릅니다.
