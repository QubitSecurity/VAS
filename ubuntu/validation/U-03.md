## [U-03] 계정 잠금 임계값 설정

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | U-03 |
| 항목명 | 계정 잠금 임계값 설정 |
| 위험도 | 상 (High) |
| 범주 | 계정 관리 |

### 점검 목적

일정 횟수 이상의 로그인 실패 시 계정을 잠그지 않으면 무차별 대입 공격이 무한히 지속될 수 있습니다. 로그인 실패 임계값을 설정하여 자동화된 패스워드 공격을 차단해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 로그인 실패 5회 이하로 계정 잠금 임계값이 설정된 경우 | 무차별 대입 공격 시 자동으로 계정 잠금 |
| 취약 | 계정 잠금 임계값이 설정되지 않거나 5회를 초과하는 경우 | 무제한 로그인 시도 가능 |

## 베이스라인 기준

**안전(양호)**

Ubuntu 20.04 (`pam_tally2` 사용):
- `/etc/pam.d/common-auth`에 `deny=5`, `unlock_time=300` 이상 설정

```
auth  required  pam_tally2.so deny=5 unlock_time=300 onerr=fail
```

Ubuntu 22.04+ (`pam_faillock` 권장):
- `/etc/pam.d/common-auth`에 `deny=5`, `unlock_time=300` 이상 설정

```
auth  required  pam_faillock.so preauth silent deny=5 unlock_time=300
auth  required  pam_faillock.so authfail deny=5 unlock_time=300
```

또는 `/etc/security/faillock.conf`:

```
deny = 5
unlock_time = 300
```

**취약**

- `/etc/pam.d/common-auth`에 `pam_faillock` 또는 `pam_tally2` 미적용
- `deny=0` (잠금 없음) 또는 미설정
