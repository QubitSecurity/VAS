## [U-88] 디스크 암호화 사용 여부 점검 (LUKS/dm-crypt)

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | U-88 |
| 항목명 | 디스크 암호화 사용 여부 점검 (LUKS/dm-crypt) |
| 위험도 | 중 (Medium) |
| 범주 | 보안 관리 |

### 점검 목적

물리적 디스크가 탈취될 경우, 암호화되지 않은 디스크는 OS 부팅 없이도 데이터를 직접 읽을 수 있습니다. 민감한 데이터를 처리하는 서버는 LUKS/dm-crypt를 이용한 디스크 암호화를 적용하여 물리적 탈취로 인한 데이터 유출을 방지해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 민감 데이터 파티션 또는 전체 디스크에 LUKS 암호화가 적용된 경우 | 디스크 탈취 시에도 데이터 보호 |
| 취약 | 디스크 암호화가 적용되지 않은 경우 | 물리적 탈취 시 데이터 즉시 노출 |

## 베이스라인 기준

**안전(양호)**

- `lsblk -f` 결과에 `crypto_LUKS` 타입 파티션 존재
- `/dev/mapper/`에 LUKS 언락된 디바이스 존재

**취약**

- LUKS 파티션 없음 (모든 파티션이 평문)
- 민감 데이터 경로(`/home`, `/var`, `/data`)가 암호화되지 않은 파티션에 위치
