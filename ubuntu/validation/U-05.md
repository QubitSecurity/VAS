## [U-05] root 이외의 UID가 '0' 금지

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | U-05 |
| 항목명 | root 이외의 UID가 '0' 금지 |
| 위험도 | 상 (High) |
| 범주 | 계정 관리 |

### 점검 목적

UID 0은 Linux에서 root 권한을 의미합니다. root 이외의 계정에 UID 0이 할당되면 해당 계정을 통해 root와 동일한 시스템 전체 제어 권한이 부여됩니다. 이는 의도치 않은 권한 부여 또는 백도어 계정 생성에 악용될 수 있습니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | root 계정만 UID 0을 가지는 경우 | UID 0 계정이 root 하나뿐 |
| 취약 | root 이외의 계정에 UID 0이 할당된 경우 | 복수의 슈퍼유저 계정 존재, 백도어 또는 설정 오류 가능성 |

## 베이스라인 기준

**안전(양호)**

- `awk -F: '($3==0)' /etc/passwd` 결과가 `root:x:0:0:...` 한 줄만 출력

**취약**

- `root` 이외의 계정에 UID 0 할당 (예: `admin:x:0:0:...`)
