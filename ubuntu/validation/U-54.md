## [U-54] Apache 파일 업로드 및 다운로드 제한

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | U-54 |
| 항목명 | Apache 파일 업로드 및 다운로드 제한 |
| 위험도 | 중 (Medium) |
| 범주 | 서비스 관리 |

### 점검 목적

파일 업로드 기능을 통해 악성 스크립트가 서버에 업로드되거나, 제한 없는 다운로드로 대용량 파일을 통한 DoS 공격이 가능합니다. 업로드 디렉토리의 실행 권한 제거 및 전송 크기 제한을 설정해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | LimitRequestBody 설정 및 업로드 디렉토리 실행 권한이 제거된 경우 | 대용량 업로드 DoS 방지, 웹쉘 실행 차단 |
| 취약 | 업로드 크기 제한이 없고 업로드 디렉토리에서 CGI 실행이 가능한 경우 | 웹쉘 업로드 및 실행, DoS 공격 가능 |

## 베이스라인 기준

**안전(양호)**

- `LimitRequestBody 10485760` (10MB 예시) 이하 설정
- 업로드 디렉토리에 `Options -ExecCGI`, PHP 실행 비활성화
- 설정 파일 경로: `/etc/apache2/apache2.conf`

**취약**

- `LimitRequestBody 0` (무제한) 또는 미설정
- 업로드 디렉토리에 실행 권한
