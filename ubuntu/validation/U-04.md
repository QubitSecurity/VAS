| 항목 | 내용 |

| 항목명 | 패스워드 파일 보호 |

| 범주 | 계정 관리 |

### 점검 목적

`/etc/passwd`에 패스워드 해시가 직접 저장될 경우(Shadow 미사용), 파일 열람 권한이 있는 모든 사용자가 해시값을 획득하여 오프라인 크래킹이 가능합니다. Shadow Password Suite를 사용하여 패스워드 해시를 `/etc/shadow`에 분리 보관하고 엄격한 권한을 설정해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | Shadow Password Suite가 적용되어 /etc/passwd의 패스워드 필드가 'x'로 표시된 경우 | 패스워드 해시가 /etc/shadow에 분리 저장, root만 접근 가능 |
| 취약 | /etc/passwd에 패스워드 해시가 직접 저장된 경우 | 일반 사용자가 패스워드 해시 획득, 오프라인 크래킹 가능 |

## 베이스라인 기준

**안전(양호)**

- `/etc/passwd` 모든 계정의 패스워드 필드가 `x`
- `/etc/shadow` 소유자: `root`, 그룹: `shadow`, 권한: `640`
- `passwd` 패키지 설치됨

**취약**

- `/etc/passwd`에 패스워드 해시(`$6$...`) 직접 존재
- `/etc/shadow` 부재 또는 권한 과다 부여
