## [U-21] /etc/shadow 파일 소유자 및 권한 설정

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | U-21 |
| 항목명 | /etc/shadow 파일 소유자 및 권한 설정 |
| 위험도 | 상 (High) |
| 범주 | 파일 및 디렉토리 관리 |

### 점검 목적

`/etc/shadow`는 패스워드 해시를 저장하는 파일입니다. 일반 사용자가 이 파일에 접근할 경우 오프라인 패스워드 크래킹이 가능하므로, 엄격한 권한을 설정해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | /etc/shadow 소유자가 root이고 other 읽기·쓰기 권한이 없는 경우 | 패스워드 해시 노출 방지 |
| 취약 | /etc/shadow에 other 읽기 이상 권한이 부여된 경우 | 패스워드 해시 노출로 오프라인 크래킹 가능 |

## 베이스라인 기준

**안전(양호)**

- 소유자: `root`
- 그룹: `shadow` (Ubuntu 기본값)
- 권한: `640`

```
-rw-r----- 1 root shadow /etc/shadow
```

> Ubuntu는 `shadow` 그룹이 `/etc/shadow`를 읽을 수 있도록 설계되어 있으므로 `root:shadow 640`이 정상 양호 기준. RedHat 기준의 `000` 적용 시 Ubuntu 일부 시스템 도구가 정상 동작하지 않을 수 있음

**취약**

- 소유자가 `root`가 아닌 경우
- 권한이 `644`, `664`, `666` 이상인 경우 (other read 이상 허용)
- 그룹이 `shadow`가 아닌 다른 그룹인 경우
