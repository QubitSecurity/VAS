## [U-69] 보안 위협에 대한 위험 해소

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | U-69 |
| 항목명 | 보안 위협에 대한 위험 해소 |
| 위험도 | 중 (Medium) |
| 범주 | 보안 관리 |

### 점검 목적

MAC(Mandatory Access Control) 보안 기능이 비활성화되어 있으면 시스템의 접근 제어 기능이 크게 약화됩니다. Ubuntu에서는 AppArmor를 Enforce 모드로 운영하여 추가적인 접근 제어 계층을 유지해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | AppArmor가 활성화되고 주요 프로파일이 enforce 모드로 운영되는 경우 | MAC 기반 추가 접근 제어 적용 |
| 취약 | AppArmor가 비활성화되거나 모든 프로파일이 complain 모드인 경우 | MAC 접근 제어 미적용, 취약점 익스플로잇 용이 |

## 베이스라인 기준

**안전(양호)**

- `aa-status` 결과에 `apparmor module is loaded.` 확인
- enforce 모드 프로파일이 다수 존재
- `systemctl is-active apparmor` 결과: `active`

**취약**

- AppArmor 비활성화 상태 (`aa-status` 오류 또는 not loaded)
- 모든 프로파일이 `complain` 모드 (enforce 미적용)
- `apparmor` 서비스 비활성화
