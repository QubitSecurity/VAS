## [U-81] 미사용 스크립트 매핑 및 불필요한 확장자 실행 제한

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | U-81 |
| 항목명 | 미사용 스크립트 매핑 및 불필요한 확장자 실행 제한 |
| 위험도 | 상 (High) |
| 범주 | 서비스 관리 |

### 점검 목적

`AddHandler`, `AddType`, `Action` 지시어로 `.cgi`, `.pl`, `.py` 등의 확장자가 실행 가능하도록 매핑되어 있으면, 해당 확장자의 파일이 업로드될 경우 즉시 서버에서 실행됩니다. 미사용 스크립트 매핑을 제거하고 업로드 경로에서의 스크립트 실행을 차단해야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 미사용 스크립트 매핑이 제거되고 업로드 경로의 스크립트 실행이 차단된 경우 | 업로드된 악성 스크립트 실행 불가 |
| 취약 | 미사용 AddHandler/AddType 설정이 존재하거나 업로드 경로에서 스크립트 실행이 가능한 경우 | 웹쉘 업로드 및 실행 가능 |

## 베이스라인 기준

**안전(양호)**

- 사용하지 않는 `AddHandler .cgi .pl .py` 주석 처리 또는 제거
- 백업·위험 확장자 차단:

```apache
<FilesMatch "\.(bak|inc|old|sql|orig)$">
    Require all denied
</FilesMatch>
```

- 업로드 디렉토리 스크립트 실행 차단

**취약**

- `AddHandler cgi-script .cgi .pl` 활성 상태
- 업로드 디렉토리에서 PHP/CGI 실행 가능
