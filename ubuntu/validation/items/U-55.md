## [U-55] Apache 웹 서비스 영역의 분리

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | U-55 |
| 항목명 | Apache 웹 서비스 영역의 분리 |
| 위험도 | 상 (High) |
| 범주 | 서비스 관리 |

### 점검 목적

웹 서비스의 루트 디렉토리가 시스템 루트(`/`) 또는 중요 디렉토리(`/etc`, `/bin`, `/home` 등)와 같거나 그 하위에 위치하면, 웹 취약점을 통해 시스템 전체 파일에 접근할 수 있는 경로가 생깁니다. 웹 서비스 영역은 시스템 영역과 명확히 분리되어야 합니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 웹 루트가 시스템 주요 디렉토리와 분리된 전용 경로에 있는 경우 | 시스템 파일 접근 차단 |
| 취약 | 웹 루트가 /, /etc, /bin 등 시스템 주요 디렉토리 하위에 설정된 경우 | 시스템 파일 노출 위험 |

## 베이스라인 기준

**안전(양호)**

- `DocumentRoot "/var/www/html"` 또는 전용 경로
- 시스템 주요 디렉토리(`/etc`, `/bin`, `/usr`, `/root` 등) 아님

**취약**

- `DocumentRoot "/"` 또는 시스템 경로 하위 설정
