## [U-14] 사용자 홈 디렉토리 소유자 및 권한 설정

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | U-14 |
| 항목명 | 사용자 홈 디렉토리 소유자 및 권한 설정 |
| 위험도 | 중 (Medium) |
| 범주 | 계정 관리 |

### 점검 목적

홈 디렉토리의 소유자가 잘못 설정되거나 권한이 과다할 경우, 다른 사용자가 해당 홈 디렉토리의 파일(`.bash_history`, `.ssh/authorized_keys` 등)에 접근하여 정보를 탈취하거나 SSH 키를 변조할 수 있습니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 홈 디렉토리 소유자가 해당 계정이며, 권한이 750 이하인 경우 | 타 사용자의 홈 디렉토리 접근 차단 |
| 취약 | 홈 디렉토리의 소유자가 다르거나 권한이 757 이상(other write 포함)인 경우 | 타 사용자가 홈 디렉토리 내용 수정 가능 |

## 베이스라인 기준

**안전(양호)**

- 홈 디렉토리 소유자 = 계정 본인
- 권한 `750` 이하 (other 권한 없음)

**취약**

- 소유자 불일치 또는 `757`, `777` 등 other write 권한 부여
