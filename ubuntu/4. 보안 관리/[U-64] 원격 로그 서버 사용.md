## [U-64] 원격 로그 서버 사용

### 개요

| 항목 | 내용 |
|------|------|
| 점검 코드 | U-64 |
| 항목명 | 원격 로그 서버 사용 |
| 위험도 | 상 (High) |
| 범주 | 보안 관리 |

### 점검 목적

시스템이 침해된 경우 공격자가 로컬 로그를 삭제하거나 변조할 수 있습니다. 원격 로그 서버(Syslog 서버)에 실시간으로 로그를 전송하면 로컬 로그 변조 후에도 침해 증적을 보존할 수 있습니다.

### 점검 기준

| 구분 | 기준 | 설명 |
|------|------|------|
| 안전 | 중앙 로그 서버로 로그가 전송되고 있는 경우 | 시스템 침해 후 로그 변조에도 증적 보존 |
| 취약 | 로컬에만 로그를 기록하는 경우 | 시스템 침해 시 로그 삭제/변조 가능 |

## 베이스라인 기준

**안전(양호)**

- `/etc/rsyslog.conf` 또는 `/etc/rsyslog.d/` 설정에 `*.* @@<로그서버IP>:514` 설정 (TCP 권장)

**취약**

- 원격 로그 서버 설정 없음
